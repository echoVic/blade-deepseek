# Orca v0.1.114

Patch release: recover more gracefully from filesystem sandbox denials in
server `command/exec` and model-visible `bash`, especially nested-workspace
`.git/index.lock` writes that macOS Seatbelt rejects outside the active write
roots.

## Changes

- **Shared sandbox-denial diagnostics**: runtime code now recognizes common
  filesystem sandbox denial messages, extracts the denied path, suggests a
  safe write root, and explains when `.git/index.lock` failures are sandbox
  scope issues instead of stale lock files.
- **Command/exec recovery**: single-shot and streaming/processId
  `command/exec` calls can request a turn-scoped filesystem write grant for
  eligible denied paths and retry the original command with the granted root.
- **Model-visible bash recovery**: shell-session-backed `bash` tool execution
  uses the same diagnostic and permission-request path before retrying with
  the granted write root.
- **Denylist guardrails**: explicitly denied writable roots remain final
  diagnostics and do not trigger an automatic retry request.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or approval response wire shapes.

This release tightens the Codex/package-3-style permission profile behavior by
making sandbox denials actionable without broadening the default write scope.

## Install

```bash

```

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.114 sh
```
