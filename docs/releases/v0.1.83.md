# Orca v0.1.83

Patch release: extend the managed `command/exec` network proxy local-network guard to resolved DNS targets.

## Changes

- **Resolved-address guard**: the proxy now checks DNS results before connecting, so hostnames that resolve to loopback, private, link-local, CGNAT, test-net, multicast, reserved, IPv6 unique-local/link-local, or IPv4-mapped blocked ranges are rejected.
- **Codex alignment**: the connection path now mirrors Codex's target-checked connector shape: policy is applied both to the requested host and to the resolved socket address.
- **Diagnostics preserved**: resolved local/private blocks return HTTP `403` with `x-proxy-error: blocked-by-policy`, keeping command/exec clients from seeing only a dropped connection.
- **Explicit local literal compatibility**: explicitly allowlisted local IP literals such as `127.0.0.1` remain usable for intentional loopback workflows.

## Compatibility

No changes to permission profile config syntax, command/exec method names, sandbox selection, proxy URL environment variables, CLI flags, JSONL event names, TUI behavior, npm package layout, or install commands.

Domain names that previously passed a deny-only network policy but resolved to non-public addresses now receive HTTP `403`.
