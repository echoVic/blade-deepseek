# Orca v0.1.75

Patch release: ThreadStore is now split into focused storage modules while preserving the existing runtime API and JSONL behavior.

## Changes

- **ThreadStore facade**: `thread_store.rs` now exports the storage API from focused submodules instead of owning the full JSONL implementation directly.
- **Storage-neutral types**: session metadata, thread summaries, pagination payloads, and the `ThreadStore` trait live in `thread_store/types.rs`.
- **Local JSONL backend**: `JsonlThreadStore`, session list/load/search/archive/delete/rename/compress behavior, and record lookup helpers live in `thread_store/local.rs`.
- **Writer boundary**: `SessionWriter`, append/rewrite helpers, locking, redaction, and JSONL/zstd IO live in `thread_store/writer.rs`.
- **Projection boundary**: persisted message-to-thread turn/item projection lives in `thread_store/projection.rs`, with pagination helpers isolated in `thread_store/pagination.rs`.
- **Live handle boundary**: `LiveThread` now has a dedicated `thread_store/live_thread.rs` module.

## Compatibility

No changes to CLI flags, server-mode method names, npm package layout, install commands, JSONL record formats, public realtime event names, or TUI display behavior.

This release is an architecture slice only. It keeps the ThreadStore API stable while making the next storage/query refactors easier to review and test.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.75 sh
```
