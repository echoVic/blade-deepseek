# Orca v0.1.101

Patch release: configurable reasoning effort and a stepwise `/model` picker in
the TUI.

## Changes

- **Reasoning effort configuration**: a new `ReasoningEffort` setting (`high` or
  `max`, default `max`) flows through the provider config and run config and is
  sent as `reasoning_effort` on DeepSeek API requests. It can be set via the
  `ORCA_REASONING_EFFORT` / `DEEPSEEK_REASONING_EFFORT` environment variables,
  the `reasoning_effort` config-file key, or CLI arguments, following the usual
  precedence chain.
- **Stepwise `/model` picker**: the TUI `/model` command is now a two-step flow —
  first pick the model, then pick the reasoning effort. Nothing is applied until
  the flow completes, and pressing `Esc` cancels the whole selection cleanly.
- **Status bar**: the status bar now shows both the active model name and the
  reasoning effort together.
- **Docs/site copy**: README config reference, help text, and site marketing
  copy refreshed to cover reasoning effort.

## Compatibility

Additive only. New optional config key `reasoning_effort` (and matching env
vars) default to `max`, preserving previous behavior. No changes to JSON
protocol method names, server event names, install commands, or npm package
layout.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.101 sh
```
