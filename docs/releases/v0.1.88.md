# Orca v0.1.88

Patch release: add automatic network permission request and retry support for
single-shot `command/exec` permission-profile network blocks.

## Changes

- **Command/exec network permission flow**: single-shot `command/exec` calls
  with a `threadId` and `permissionProfile` now convert eligible managed proxy
  blocks into a `permission_request` event instead of completing immediately.
- **Session-scoped retry**: when the client responds with `permission/respond`
  `allow` and `scope: "session"`, Orca persists the network domain grant,
  updates the in-memory server thread, and retries the original command with
  the expanded proxy policy.
- **Structured proxy block reporting**: the managed network proxy can now report
  blocked hosts to the server directly, so permission prompts do not depend on
  parsing command stdout.
- **Denylist safety**: explicit denylist blocks stay final proxy diagnostics and
  are not escalated into permission requests.

## Compatibility

No changes to command/exec method names, permission profile TOML shape, sandbox
policy names, proxy environment variables, TUI behavior, npm package layout, or
install commands.

The automatic request/retry path currently applies to single-shot server
`command/exec` requests with a `threadId`. Streaming or TTY command processes
continue to use the existing process lifecycle.



```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.88 sh
```
