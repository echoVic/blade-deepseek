# Orca v0.1.80

Patch release: move the TUI conversation session onto `RuntimeThread`, completing the first server/headless/TUI convergence pass for long-lived runtime-owned agent state.

## Changes

- **TUI RuntimeThread adoption**: `TuiConversationSession` now owns `RuntimeThread` instead of directly owning `InteractiveSession` and `RuntimeSessionLifecycle`.
- **Preloaded resume/fork path preserved**: `RuntimeThread::start_with_preloaded` gives TUI startup the same preloaded transcript behavior it had before while moving ownership behind the runtime boundary.
- **Runtime lifecycle continuity**: TUI turn-start task lifecycle events still come from `RuntimeTurnRunner`, but the lifecycle now lives inside `RuntimeThread`.
- **Architecture guard**: a new TUI bridge test prevents the session wrapper from reintroducing direct `InteractiveSession` or `RuntimeSessionLifecycle` ownership.

## Compatibility

No changes to TUI keybindings, slash commands, approval prompts, workflow controls, goal mode, JSONL history formats, CLI flags, server protocol method names, npm package layout, or install commands.

This release is an architecture slice only. It narrows the TUI bridge ownership surface while preserving conversation reuse, final assistant rendering, turn lifecycle events, goal tool exposure, workflow state, request-user-input continuation, and streaming bash cancellation behavior.



```bash
curl -fsSL https://orcaagent.dev/install.sh | \
```
