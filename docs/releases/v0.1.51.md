# Orca v0.1.51

Patch release: MCP registry startup errors in resource discovery.

## Changes

- **Registry-level MCP errors**: all-server `list_mcp_resources` results now
  include MCP registry initialization errors, so failed server startup remains
  visible next to resources from healthy servers.
- **Template discovery parity**: all-server `list_mcp_resource_templates`
  results now include the same registry-level initialization errors alongside
  per-server `resources/templates/list` failures.
- **Strict server filters preserved**: specifying `server` still targets only
  that server and returns the strict resource/template array or that server's
  error. Unrelated registry startup failures are not mixed into filtered calls.
- **Test utility coverage**: MCP registry tests and tool executor tests now
  cover registry startup errors separately from per-server list failures.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode wire
shapes, npm package layout, install commands, MCP tool routing, existing
`read_mcp_resource`, or filtered `list_mcp_resources` /
`list_mcp_resource_templates` behavior. The all-server `errors` arrays may now
include registry startup failures in addition to per-server list failures.
