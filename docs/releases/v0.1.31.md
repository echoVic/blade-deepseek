# Orca v0.1.31

Patch release: P0 session runtime unification.

## Changes

- **Runtime-owned interactive sessions**: `orca-runtime` now owns the interactive conversation, history writer, session id, project instructions, memory, hooks, MCP registry, cost tracking, and workflow task registry through `InteractiveSession`.
- **TUI compatibility wrapper**: `orca-tui` keeps `TuiConversationSession` as a narrow wrapper while delegating session state to runtime, preserving current display and user-action behavior.
- **History and compaction boundary**: resume/fork replay, history append/complete, backtracking, pinned context, goal context, skill context, and local compaction now flow through the runtime-owned session.
- **Workflow state consistency**: active workflow detection now reads the runtime task registry, reducing drift between TUI and runtime workflow state.
- **Architecture docs**: the production roadmap now treats P0 as the runtime session boundary and records P1 as the protocol/event layer follow-up.

## Compatibility

This release does not change public TUI event names, JSONL event names, CLI flags,
tool names, or install commands.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash

```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.31 sh
```
