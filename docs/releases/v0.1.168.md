# Orca v0.1.168

Patch release: move tool-dispatch cursor ownership into the runtime-owned
sampling request state.

## Changes

- **Request-state cursor ownership**: `RuntimeSamplingRequestState` now owns the
  current tool-request cursor in addition to the per-sampling permission
  overlay.
- **Tool-turn dispatch cleanup**: `tool_turn` reads and advances the current
  request through sampling state instead of keeping a local `ToolRequestCursor`.
- **Batch advancement boundary**: readonly and subagent batch dispatch advance
  the sampling-state cursor through one clamped helper, keeping single-tool and
  batch cursor movement behind the same request-state API.
- **Regression coverage**: the runtime architecture contracts now require tool
  cursor state to live in `step_context.rs`, require tool turns to use the
  sampling-state cursor APIs, and reject a production `ToolRequestCursor`.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice. Provider responses, normal tool
execution, permission requests, readonly batching, subagent batching, workflows,
history projection, and public release packaging keep the same user-visible
behavior as v0.1.167.

```bash
npm install -g @blade-ai/orca

```


```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.168 sh
```
