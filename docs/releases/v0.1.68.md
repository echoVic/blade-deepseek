# Orca v0.1.68

Patch release: TUI tool approval gating is now owned by the runtime interaction
adapter instead of the TUI bridge.

## Changes

- **Approval gate adapter**: TUI approval request construction, approval policy
  resolution, preview generation, and interactive approval waiting now live in
  `crates/orca-tui/src/runtime_interaction_adapter.rs`.
- **Thinner TUI bridge**: `bridge.rs` delegates approval gating to the adapter
  and stays focused on orchestration and tool result/event completion.
- **Behavior-preserving tests**: ownership tests prevent the approval gate from
  drifting back into `bridge.rs`, while approval approve/cancel tests preserve
  existing runtime handler behavior.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, public realtime event names, or TUI
display behavior.

## Install

```bash
npm install -g @blade-ai/orca

```

Direct binary install:

```bash

```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.68 sh
```
