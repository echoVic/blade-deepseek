# Orca v0.1.181

Patch release: assemble turn-loop state through the runtime turn kernel.

## Changes

- **Kernel-owned loop-state assembly**: `RuntimeTurnKernel` now assembles the
  lifecycle-owned `RuntimeTurnLoopState` used by the runtime turn loop.
- **Narrower lifecycle handoff**: `RuntimeTurnState` no longer expands
  directive state, mutable runtime refs, and extension-state fields into the
  loop state itself.
- **Extension state constructor**: `RuntimeTurnExtensionState` now exposes a
  focused internal constructor so extension registry/thread/turn stores stay
  grouped without making its fields public.
- **Architecture guardrail**: a focused runtime test prevents turn-loop state
  assembly from drifting back into `RuntimeTurnState`.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, or JSONL event names.

Runtime turn setup, provider response handling, tool dispatch, approval
overlays, workflow tools, subagent batching, goal progress, and history
recording remain behavior preserving.
