# Orca v0.1.94

Patch release: move server turn-control dispatch into a focused server
processor module.

## Changes

- **Turn processor boundary**: `server/router.rs` now delegates decoded
  turn-control operations to `server/processors/turn.rs`.
- **Covered operations**: `TurnInterrupt`, `TurnResume`, and `TurnSteer` now
  share the focused turn processor entrypoint.
- **Architecture guardrail**: a new runtime architecture test keeps turn
  control dispatch details out of the generic router.
- **Behavior preservation**: active-turn interruption, resume, steer input
  injection, turn/thread mismatch errors, idle controls, and completed-turn
  rejection keep the same server events and semantics.

## Compatibility

No changes to CLI flags, JSON protocol methods, server event names, permission
profile TOML shape, TUI behavior, install commands, or npm package layout.

This release continues the Codex-style request-processor refactor path. Submit,
permission, shell, and command/exec processor extractions remain follow-up
slices.
