# Orca v0.2.23

Feature release: native-feeling mouse text selection and clipboard copy in the
TUI, with edge auto-scroll, double-click word selection, and a floating
"Jump to bottom" control.

## Why

Orca's fullscreen TUI enables terminal mouse capture so the wheel can scroll
the conversation, which disables the terminal emulator's own drag selection.
Copying transcript text required the modifier-drag escape hatch
(`shift`/`option` + drag) advertised on the status line. This release
implements selection and clipboard writing inside the TUI itself, so plain
left-drag works the way it does in Claude Code and desktop editors while the
wheel keeps scrolling the conversation.

## Changes

- **Drag selection over the transcript**: left press anchors a selection in
  content space (absolute wrapped-row plus display column, the same space as
  the transcript render cache), the drag grows it, and clicks or presses
  outside the transcript dismiss it. Because anchors live in content space,
  streaming output and scrolling never shift what was selected. Highlighting
  is applied to materialized rows at render time with a per-theme
  `selection_bg` color that preserves each span's foreground, and the
  selection-agnostic render cache is never invalidated by it.
- **Copy on release via OSC 52**: releasing the button extracts the selected
  text and stages it for the app loop, which writes an OSC 52 escape sequence
  to stdout (works in VS Code, iTerm2, kitty, WezTerm, and over SSH) and
  additionally pipes through `pbcopy` on macOS. A transient
  `copied N chars to clipboard` notice shows on the status line for a few
  seconds; its expiry drives one final redraw so it removes itself even while
  the session is idle.
- **Wrap-aware text extraction**: the word wrapper now records the whitespace
  it drops at each soft-wrap boundary (`wrap_gaps`), so soft-wrapped prose
  copies back as one line with its spacing restored while hard-split long
  words such as URLs stay unbroken. Hard line and message boundaries emit
  newlines with trailing whitespace trimmed. Column slicing counts display
  width, so CJK and other wide characters select and copy correctly.
- **Double-click word selection**: a second press on the same cell within
  400ms selects the same-class run (alphanumeric/underscore, punctuation, or
  whitespace) under the cursor and copies it immediately.
- **Edge auto-scroll**: dragging onto the transcript's first or last row
  scrolls immediately and arms a direction that the 80ms animation tick keeps
  applying, so the selection keeps growing while the pointer sits still —
  terminals only deliver drag events on movement. Moving back inside,
  releasing, or pressing again disarms it.
- **Jump to bottom**: when auto-follow is disarmed by scrolling up, a floating
  `Jump to bottom (click) ↓` pill appears centered over the transcript's
  bottom row. Clicking it re-arms follow and consumes the click without
  starting a selection; the click target is re-registered every frame and
  cleared on panel switches so no stale region can catch clicks.
- **Status line cleanup**: the obsolete `shift+drag to copy` hint is removed.
  Wheel scrolling, scroll batching, and follow re-arming behavior are
  unchanged.

## Verification

- `cargo test -p orca-tui --lib` — 452 tests pass, including new coverage for
  selection normalization and column math, wide-character slicing, OSC 52
  encoding, span splitting with foreground preservation, wrap-gap restoration,
  hard-split long words, multi-message extraction, word selection classes,
  the drag/click/double-click/edge-scroll state machine, tick-driven parked
  auto-scroll, copy-notice display and expiry, and the jump-pill
  appear/click/leave lifecycle.
- Compact wrapper parity tests against ratatui's `WordWrapper` continue to
  pass with `wrap_gaps` recording in place.
- Manual smoke test in VS Code's terminal: drag select with highlight, release
  to copy, double-click word copy, top/bottom edge auto-scroll with a parked
  pointer, and jump-to-bottom re-arming follow.
