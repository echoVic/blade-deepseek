# Orca v0.1.33

Patch release: P2 tool system convergence.

## Changes

- **Runtime tool invocation boundary**: new `orca_runtime::tool_invocation`
  centralizes requested/effective tool records, canonical action derivation,
  approval request construction, and schema-validation failures.
- **Shared approval path**: controller and TUI approval prompts now use the same
  runtime helper, so caller-supplied `ToolRequest.action` cannot downgrade shell
  or write-capable tools.
- **Hook-modified validation**: pre-tool hook `modified_target` output is
  reapplied through the shared invocation path and revalidated before execution.
- **Batch paths aligned**: readonly and subagent batch execution now route
  validation through the same runtime boundary while keeping execution ownership
  in the controller.
- **MCP test utility isolated**: MCP registry test construction is behind an
  explicit `test-utils` feature instead of expanding the normal public API.

## Compatibility

This release does not change public tool names, external tool TOML descriptors,
MCP tool names, JSONL event names, server-mode flat event mapping, or TUI
approval dialog behavior.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.33 sh
```
