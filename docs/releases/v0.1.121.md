# Orca v0.1.121

Patch release: continue the P0 runtime reducer split by moving steer
application out of `lifecycle.rs` and into a focused `runtime_steer` module
with a grouped input boundary.

## Changes

- **Dedicated steer module**: `RuntimeSteerStep` now lives in
  `runtime_steer.rs` instead of `lifecycle.rs`.
- **Grouped steer input**: `RuntimeSteerInput` groups the optional
  `ThreadSteerHandle`, mutable conversation, and optional history writer so the
  extracted boundary does not preserve the old multi-argument call surface.
- **Preserved turn behavior**: `RuntimeTurnOpeningStep` and
  `RuntimeProviderTurnStep` still drain pending steer inputs before model calls,
  add them as user messages, and append them to history when a writer is
  present.
- **Smaller lifecycle surface**: `lifecycle.rs` keeps `ThreadSteerHandle`
  storage and surrounding actor/lifecycle primitives while shedding another
  reducer-sized orchestration slice.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, active
`turn/steer` item streaming, multi-text steer injection, conversation/history
recording, model routing, compaction, tool execution, approvals, workflows,
shell sessions, and history persistence remain unchanged.

