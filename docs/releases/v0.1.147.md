# Orca v0.1.147

Patch release: group child-agent runtime constructor inputs behind a focused
context object so the child-agent runtime API keeps shrinking without changing
runtime behavior.

## Changes

- **Grouped runtime context**: `ChildAgentRuntimeContext` now carries cwd,
  event factory, event sink, project instructions, memory block, MCP registry,
  hooks, cancel token, optional lifecycle, and child executor for
  `ChildAgentRuntime::new`.
- **Shorter runtime constructor**: `ChildAgentRuntime::new` now accepts the
  grouped context instead of a long runtime-environment argument list.
- **Facade compatibility**: `agent_child.rs` re-exports
  `ChildAgentRuntimeContext` alongside the existing child-agent runtime types so
  callers can keep using the compatibility facade.
- **Ownership coverage**: the child-agent shared types architecture test now
  locks the grouped runtime context boundary and rejects the previous long
  constructor signature.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, provider turn dispatch, provider hook execution, child compaction,
prompt-too-long recovery, response folding, tool-result folding, cost tracking,
approval policy semantics, MCP routing, external tool routing, shell session
protocol, command/exec wire shape, conversation storage, history storage, npm
package layout, or install commands.

This release is an internal ownership and API-shape slice. Existing runtime
callers can continue importing child-agent runtime pieces through
`orca_runtime::agent_child`, while the runtime constructor no longer creates a
new long-argument surface.
