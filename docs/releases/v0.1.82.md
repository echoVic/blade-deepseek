# Orca v0.1.82

Patch release: tighten the managed `command/exec` network proxy around local and private network targets.

## Changes

- **Local/private target guard**: permission-profile network proxy traffic now blocks `localhost`, loopback IPs, private IPv4 ranges, link-local ranges, carrier-grade NAT ranges, unique-local IPv6, and IPv6 link-local literals unless the host is explicitly allowlisted.
- **Allowlist exception preserved**: existing workflows that intentionally allow `127.0.0.1` or another local literal in `[permission_profiles.<name>.network.domains]` continue to proxy successfully.
- **Codex alignment**: the proxy now mirrors the first local-network safety layer from Codex's network proxy while keeping Orca's current synchronous loopback HTTP proxy architecture.

## Compatibility

No changes to permission profile config syntax, command/exec method names, sandbox selection, proxy URL environment variables, CLI flags, JSONL event names, TUI behavior, npm package layout, or install commands.

Requests to local/private literal targets that were only implicitly allowed by deny-only domain policies now receive HTTP `403` unless the target is explicitly allowlisted.
