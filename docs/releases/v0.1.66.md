# Orca v0.1.66

Patch release: TUI runtime event projection now has a dedicated module.

## Changes

- **Dedicated projection boundary**: `EventEnvelope -> TuiEvent` mapping now
  lives in `crates/orca-tui/src/runtime_event_projection.rs` instead of
  `bridge.rs`.
- **Thinner TUI bridge**: `bridge.rs` now imports the runtime-event projection
  helper and stays focused on session, tool, approval, subagent, and workflow
  orchestration.
- **Workflow notification shaping**: workflow terminal notification prompt
  construction moved with the projection logic, keeping runtime event payload
  display rules in one place.
- **Regression coverage**: projection behavior tests moved with the new module,
  and an ownership guard prevents the adapter from drifting back into
  `bridge.rs`.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, public realtime event names, or TUI
display behavior.
