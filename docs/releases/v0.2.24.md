# Orca v0.2.24

This release makes every accepted tool invocation end truthfully, bounds
process and file admission, and gives long-running runtime resources explicit
cleanup ownership.

Status: released on 2026-07-15. The complete local and live-provider gates are
recorded below; the post-publish verifier confirms the public GitHub and npm
artifacts after publication.

## Incident Boundary

The reported v0.2.18 Goal incident was not an Orca transcript heap leak. Ten
orphaned Vitest/Tinypool workers retained 40.51 GiB after macOS Seatbelt denied
their parent's cleanup signals. v0.2.19 fixed that direct cause by allowing a
sandboxed process to signal its own descendants. The v0.2.24 hardening is
separate defense in depth for other ownership, output, persistence, and file
admission boundaries.

The evidence is documented in
[`docs/reports/2026-07-12-goal-resource-incident.md`](../reports/2026-07-12-goal-resource-incident.md).

## Changes

- **Truthful invocation terminals**: every accepted tool call receives one
  terminal result, including interruption, pre-execution rejection, and
  not-started siblings. Legacy or crash-truncated history is repaired as
  `indeterminate` without re-executing the old call.
- **Stable projections**: persisted tool results and TUI/server events preserve
  call identity, terminal source, and whether execution started. Live thread
  state wins over stale disk projection while a turn is still running.
- **Bounded process ingress**: ordinary captured stdout and stderr retain at
  most 1 MiB per stream with exact omitted-byte accounting. Streaming shell,
  verifier, external-tool, and grep paths share bounded collectors.
- **Bounded file admission**: reads stream through regular-file, UTF-8, growth,
  cancellation, and output limits. Exact edits reject files above 16 MiB before
  mutation, while edit/write previews bound input, output, and computation
  without rereading the workspace from the TUI.
- **Owned subprocesses**: external tools, stdio MCP servers, workflows, async
  subagents, verifier commands, server turns, ordinary shells, and search
  managers terminate, wait, or hand remaining cleanup to an explicit reaper.
- **Bounded transports**: MCP stdio frames and response handoff, WorkflowHost
  events and total event bytes, server shutdown phases, and internal worker key
  handoff all have explicit limits.
- **Completion wins races**: a process observed complete after a cancellation
  callback or at a timeout boundary remains completed rather than being
  relabeled as cancelled or timed out.

## Compatibility

CLI flags, slash commands, approval modes, Goal records, provider and MCP
configuration, npm package names, unified Mention targets, and public tool
arguments remain compatible. Tool persistence and server events gain optional
terminal metadata; older records remain readable and missing terminals are
repaired without rewriting source JSONL. Unix builds terminate owned process
groups; non-Unix builds retain direct-child kill and wait behavior.

## Remaining Risk

- Windows does not yet guarantee descendant-tree termination beyond the direct
  child.
- WorkflowHost has bounded events and per-step cleanup but no total workflow
  execution deadline.
- The managed network proxy has I/O deadlines but no connection-count ceiling
  or unified owner that joins every connection worker.
