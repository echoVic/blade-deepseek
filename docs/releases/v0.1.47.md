# Orca v0.1.47

Patch release: MCP resource list/read tools.

## Changes

- **MCP resources protocol support**: stdio and SSE MCP transports can now call
  `resources/list` and `resources/read` alongside the existing tool APIs.
- **Read-only resource tools**: models can use `list_mcp_resources` and
  `read_mcp_resource` to inspect resources exposed by configured MCP servers.
- **Shared registry semantics**: the new tools use the same `ToolSpec`
  capability metadata, schema validation, read-only action kind, and
  concurrency-safe execution path as other built-in read tools.
- **Execution path coverage**: tests now prove both the low-level MCP resource
  client flow and the high-level `execute_with_mcp` resource-tool path.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode wire
shapes, npm package layout, install commands, or existing MCP tool routing.
This release adds optional read-only MCP resource access for servers that expose
the MCP resources API.


