# Orca v0.1.119

Patch release: continue the P0 runtime reducer split by moving turn-start
orchestration out of `lifecycle.rs` and into a focused `runtime_turn_start`
module.

## Changes

- **Dedicated turn-start module**: `RuntimeTurnStartStep`,
  `RuntimeTurnStartResultStep`, `RuntimeTurnStartStepOutput`, and
  `RuntimeTurnStartResult` now live in `runtime_turn_start.rs` instead of
  `lifecycle.rs`.
- **Preserved opening composition**: `RuntimeTurnOpeningStep` still composes
  compaction, turn start, model routing, and steering in the same order, but it
  now imports turn-start orchestration through the focused module.
- **Smaller lifecycle surface**: `lifecycle.rs` keeps the actor, lifecycle
  state, `RuntimeTurnStartError`, model-route, and steer primitives while
  shedding another lower-level reducer slice.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, turn-start
events, max-turn handling, model routing, compaction, steering, tool execution,
approvals, workflows, shell sessions, and history persistence remain unchanged.
