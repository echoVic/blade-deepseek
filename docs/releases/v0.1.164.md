# Orca v0.1.164

Patch release: move the turn-loop runtime handoff behind a lifecycle-owned
`RuntimeTurnLoopState` boundary.

## Changes

- **Loop runtime state handoff**: `RuntimeTurnState` now converts into
  `RuntimeTurnLoopState`, which separates runtime directives from the grouped
  mutable loop refs used by `RuntimeTurnLoopInput`.
- **No extension-field destructuring in agent_loop**: `agent_loop` no longer
  destructures `extension_registry`, `thread_extensions`, or `turn_extensions`,
  and no longer calls `extension_context_from_parts(...)`.
- **Iteration-owned extension context**: `RuntimeTurnLoopInput` now owns the
  grouped loop runtime state and derives `RuntimeExtensionContext` at the
  iteration boundary before handing it to provider turns.
- **Codex-style state direction**: this follows the Codex/package-3 reference
  shape where turn-scoped runtime state is owned by the lifecycle boundary and
  consumed by the loop as a grouped state object instead of as parallel fields.
- **Regression coverage**: architecture tests now reject extension raw-part
  reconstruction inside production `agent_loop` and verify the turn loop derives
  grouped extension context from lifecycle-owned runtime state.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice. Provider turns, normal tool lifecycle
contributors, goal progress recording, cost accounting, cancellation, task
registry access, and runtime directive behavior keep the same user-visible
semantics as v0.1.163.
