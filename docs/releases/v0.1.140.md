# Orca v0.1.140

Patch release: isolate the child-agent response-folding boundary so the
`agent_child` module keeps compatibility re-exports while provider response
folding, tool request extraction, tool execution context, and tool-result
folding live in a focused module.

## Changes

- **Focused response-folding module**: `child_agent_response_folding.rs` now
  owns `ChildAgentProviderResponseFold`, `ChildAgentToolResultFold`,
  `ChildAgentToolExecution`, `ChildAgentToolContext`,
  `fold_child_agent_provider_response`, `child_agent_tool_requests`, and
  `fold_child_agent_tool_result`.
- **Compatibility re-export**: existing `orca_runtime::agent_child` imports for
  child-agent folding APIs remain available via re-export.
- **Agent-child shrink**: `agent_child.rs` no longer owns provider response
  usage folding, assistant-message folding, provider tool-call extraction,
  tool-result model-context formatting, nested child-cost merging, or
  stop/failure folding.
- **Ownership coverage**: a new architecture test locks the response-folding
  boundary so the `agent_child` facade does not silently reacquire folding
  details.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, provider turn dispatch, provider hook execution, child compaction,
prompt-too-long recovery, cost tracking, approval policy semantics, MCP routing,
external tool routing, shell session protocol, command/exec wire shape,
conversation storage, history storage, npm package layout, or install commands.

This release is an internal ownership slice. Child-agent behavior for subagent
tool calls, workflow child agents, async subagents, TUI child-agent delegation,
provider bootstrap, model routing, provider turns, tool-call extraction,
tool-result folding, nested child-cost tracking, and turn-budget exhaustion is
unchanged; the response-folding boundary is smaller and easier to keep aligned
with the rest of the Codex/package-3-inspired runtime split.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.140 sh
```
