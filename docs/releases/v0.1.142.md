# Orca v0.1.142

Patch release: isolate the shared child-agent type boundary so the
`agent_child` module keeps compatibility re-exports while child request,
result, runtime, and executor types live in a focused module.

## Changes

- **Focused shared-types module**: `child_agent_types.rs` now owns
  `ChildAgentRequest`, `ChildAgentResult`, `ChildAgentExecutor`, and
  `ChildAgentRuntime`.
- **Compatibility re-export**: existing `orca_runtime::agent_child` imports for
  child-agent request/result types and crate-internal runtime/executor types
  remain available via re-export.
- **Focused module dependencies**: child-agent loop setup, provider turn,
  response folding, and loop runner modules now import shared child-agent types
  from `child_agent_types` directly instead of depending on the facade.
- **Ownership coverage**: a new architecture test locks the shared type
  boundary so the `agent_child` facade does not silently reacquire those types.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, provider turn dispatch, provider hook execution, child compaction,
prompt-too-long recovery, response folding, tool-result folding, cost tracking,
approval policy semantics, MCP routing, external tool routing, shell session
protocol, command/exec wire shape, conversation storage, history storage, npm
package layout, or install commands.

This release is an internal ownership slice. Child-agent behavior for subagent
tool calls, workflow child agents, async subagents, TUI child-agent delegation,
provider bootstrap, model routing, provider turns, tool-call extraction,
tool-result folding, nested child-cost tracking, and turn-budget exhaustion is
unchanged; the shared type boundary is now smaller and easier to keep aligned
with the rest of the Codex/package-3-inspired runtime split.

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.142 sh
