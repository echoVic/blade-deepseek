# Orca v0.1.54

Patch release: shared tool item projection helpers.

## Changes

- **Shared projection boundary**: realtime app-server streams and persisted
  thread projections now use `tool_item_projection` for MCP tool-name parsing,
  JSON argument parsing, MCP result shaping, and camelCase tool error objects.
- **Schema drift guardrail**: a new ownership test prevents `server_runtime`
  and `thread_store` from reintroducing separate copies of the shared MCP item
  projection helpers.
- **Projection behavior coverage**: shared helper tests now lock structured MCP
  result preservation and `error.exitCode` naming.
- **Realtime/history compatibility**: existing `mcpToolCall` and
  `dynamicToolCall` item payloads remain wire-compatible while the duplicated
  projection code is reduced.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode
methods, npm package layout, install commands, or public item field names. This
release is an internal projection-boundary refactor with regression coverage for
the public payloads.
