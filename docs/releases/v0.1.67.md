# Orca v0.1.67

Patch release: TUI runtime interaction adapters now have a dedicated module,
and the website prerender pipeline includes its server entrypoint.

## Changes

- **Dedicated interaction adapter boundary**: TUI runtime approval and
  `request_user_input` handlers now live in
  `crates/orca-tui/src/runtime_interaction_adapter.rs` instead of
  `bridge.rs`.
- **Thinner TUI bridge**: `bridge.rs` imports the interaction adapters and
  stays focused on session, tool, subagent, workflow, and event orchestration.
- **Behavior-preserving tests**: approval allow/deny and user-input
  answer/cancel tests moved with the adapter module, with an ownership guard to
  keep the handlers from drifting back into `bridge.rs`.
- **Site prerender completeness**: the server-render entry used by the site
  prerender script is tracked with the release so `npm --prefix site run build`
  works in a clean checkout.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, public realtime event names, or TUI
display behavior.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.67 sh
```
