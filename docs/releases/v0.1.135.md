# Orca v0.1.135

Patch release: isolate the runtime `request_user_input` boundary so lifecycle
coordination keeps delegating while user-input request parsing and handler
dispatch live in a focused module.

## Changes

- **Focused runtime user-input module**: `runtime_user_input.rs` now owns
  `RuntimeUserInputRequest`, `RuntimeUserInputHandler`, JSON argument parsing,
  and the helper that maps handler answers or cancellations into `ToolResult`s.
- **Compatibility re-export**: existing `orca_runtime::lifecycle` imports for
  `RuntimeUserInputRequest` and `RuntimeUserInputHandler` remain available via
  re-export.
- **Lifecycle delegation**: `RuntimeTaskActor::execute_user_input_tool` now
  delegates to the focused module instead of owning request parsing inline.
- **Ownership coverage**: a new architecture test locks the module boundary so
  `lifecycle.rs` does not silently reacquire the user-input parsing and dispatch
  details.

## Compatibility

No changes to CLI flags, JSONL event names, `request_user_input` tool schema,
answer/cancel result text, bash tool input/output shape, shell session protocol,
command/exec wire shape, permission-profile configuration, network allow/deny
semantics, filesystem sandbox grants, Unix socket allowlist semantics, MCP
routing, external tool routing, conversation storage, history storage,
workflows, TUI rendering, npm package layout, or install commands.

This release is an internal ownership slice. Runtime and TUI behavior for
`request_user_input` is unchanged; the boundary is smaller and easier to keep
aligned with the rest of the Codex/package-3-inspired runtime split.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.135 sh
```
