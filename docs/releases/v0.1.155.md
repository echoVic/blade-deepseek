# Orca v0.1.155

Patch release: route completed-tool goal progress through a runtime-owned turn
reducer before TUI state can affect terminal goal updates.

## Changes

- **Runtime turn reducer**: `orca_runtime::runtime_state` now exposes
  `RuntimeTurnReducer` and `RuntimeToolFinish` as the runtime-owned boundary for
  completed-tool state reduction.
- **Tool progress state grouping**: completed-tool progress is delegated through
  `ToolRuntimeState`, keeping the goal-progress write behind the reducer instead
  of leaking the helper into clients.
- **TUI reducer routing**: TUI completed normal-tool recording now constructs a
  runtime turn reducer over the live thread and turn extension stores instead
  of calling the goal-progress helper directly.
- **Regression coverage**: tests prove the reducer records completed-tool goal
  progress and that the TUI guard refuses direct goal-progress writes from the
  agent runner.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, TUI display
layout, server wire shapes, app-server events, JSONL event names, thread
history, goal storage format, tool schemas, npm package layout, or install
commands.

This is an internal architecture slice that follows the Codex/package-3
direction: runtime owns state reduction, while TUI stays a client of runtime
state. It prepares the next reducer slices for directive, permission, and
turn-control state without changing user-facing behavior.



