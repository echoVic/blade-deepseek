# Orca v0.2.15

Patch release: make TUI session resume reject incomplete DeepSeek assistant
turns before history replay.

## Changes

- **Shared assistant validity rule**: an assistant turn is replayable only when
  it contains visible content or at least one tool call. Reasoning metadata by
  itself is not a complete assistant response.
- **Provider retry**: DeepSeek responses that contain reasoning but no visible
  content or tool calls use the existing empty-response retry path instead of
  entering the conversation.
- **Persistence guard**: invalid assistant responses are rejected before the
  conversation or JSONL history is mutated.
- **Resume repair**: TUI and headless resume normalization remove legacy
  reasoning-only assistant turns before the next provider request.
- **Tool-call reasoning preserved**: reasoning attached to a valid tool-call
  turn remains in history and is replayed to DeepSeek.
- **Real API regression gate**: the release smoke test now seeds a malformed
  history transcript, resumes it with the real DeepSeek API, and verifies a
  successful sentinel response.

## TUI impact

Resuming an older session can no longer send an incomplete reasoning-only
assistant turn back to DeepSeek. Users see the resumed conversation continue
normally instead of failing because a partially persisted response violates the
provider's replay contract.

The repair is in-memory during resume. Existing JSONL files remain readable and
are not destructively rewritten.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, approval options, shortcuts, or valid history records.
