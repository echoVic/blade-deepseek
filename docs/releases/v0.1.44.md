# Orca v0.1.44

Patch release: fuzzy model-facing file discovery.

## Changes

- **Fuzzy `glob` mode**: the model-facing `glob` tool now accepts
  `{"mode":"fuzzy","query":"..."}` for path discovery when the model knows
  fragments or initials rather than a precise glob pattern.
- **Compatible glob behavior**: existing `{"pattern":"**/*.rs"}` calls keep the
  same default behavior, output shape, truncation handling, and `(no matches)`
  result.
- **Workspace-relative ranking**: fuzzy results are returned as
  workspace-relative file and directory paths, skip hidden entries, and are
  ranked by match quality, path length, and path text.
- **Schema-level contract**: the canonical tool registry exposes `mode`,
  `query`, and `oneOf` metadata so provider schemas advertise the new fuzzy
  input shape alongside existing runtime argument checks.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm package
layout, install commands, or existing glob/list-files behavior.
