# Orca v0.1.127

Patch release: continue the Codex-style server boundary split by moving
server active-turn lifecycle state out of `server.rs` and into a focused
`server/active_turn_manager.rs` module.

## Changes

- **Dedicated active turn manager module**: `ActiveTurnManager`,
  `ActiveTurnControl`, and `ActiveTurnHandle` now live in
  `server/active_turn_manager.rs`, where they own active turn controls,
  running turn handles, finished-turn reclamation, thread-specific reclaim
  waits, and completed-turn metadata merge for session-scoped permission
  grants.
- **Smaller server orchestrator**: `server.rs` keeps stdio orchestration,
  permission response event emission, submit startup, and turn-control event
  shaping while delegating active turn bookkeeping to the manager.
- **Behavior-compatible active turns**: interrupt, resume, steer, idle
  controls, completed-turn rejection, thread mismatch handling, active turn ids,
  and session permission metadata persistence keep the same server wire shape.
- **Documented P0 slice**: added a focused implementation plan for the active
  turn manager boundary so the next server lifecycle split can continue from a
  smaller `server.rs`.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, turn ids, thread ids, permission response
wire shapes, or turn-control semantics.

This release is an internal architecture slice. Server-mode turn execution,
command/exec, shell sessions, workflows, subagents, history projection, and
server/TUI/headless runtime execution remain unchanged.
