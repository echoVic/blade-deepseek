# Orca v0.2.33

Feature release: keep every submitted TUI or server prompt as exactly one user
row through live use, pagination, restart, and resume.

## Why

Hosted turn preparation already admitted and persisted the user prompt. The
agent-loop bootstrap then received the same borrowed conversation and writer
through ambiguous optional fields and persisted the final user message again.
The model saw one prompt, but durable history contained two records with the
same turn id and different item ids. Live and cold thread APIs exposed both.

This was a write-ownership defect, not a projection defect. Hiding duplicates
while reading would leave two conflicting durable facts and make pagination,
search, and future reducers depend on special-case suppression.

## Changes

- **One hosted admission owner**: `ThreadTurnContext` is the only boundary that
  adds and persists a submitted hosted prompt.
- **Transactional prompt admission**: the identified user record must append
  successfully before the prompt enters model context or public projection.
- **Typed conversation provenance**: agent-loop bootstrap explicitly receives
  either an owned child-agent conversation or a borrowed runtime session.
- **No borrowed history replay**: borrowed sessions retain their writer for
  later tool and completed-model records without bootstrapping the prompt a
  second time.
- **Stable live and cold history**: `thread/read`, `thread/turns/list`, and
  `thread/items/list` return one user item per logical turn before and after
  server exit, restart, pagination, and resume.
- **Old path deleted**: the duplicate initial-history helper and source-shape
  ownership tests are removed; no read-time deduplication layer replaces them.
- **Real DeepSeek proof**: the release gate verifies exact user counts and
  stable turn/item identities across live and cold cross-process projections.

## Compatibility

CLI arguments, TUI keys and flows, server methods and JSONL response wrappers,
DeepSeek model-facing conversation content, and persistence formats remain
compatible. Newly recorded hosted turns contain one user record instead of two.

Existing histories that already contain duplicate user records remain readable
and are not rewritten during normal read, resume, compaction, archive, or
compression. Cleaning those historical identities would require an explicit
migration and is not hidden behind ordinary projection.

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.33
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.33 sh
```
