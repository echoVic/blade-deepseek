# Orca v0.1.36

Patch release: workflow isolation and recovery.

## Changes

- **Workflow agent worktrees**: workflow agents can now opt into isolated git
  worktrees, keeping concurrent runs from stepping on each other.
- **Async subagent recovery**: async subagent handles are persisted so later
  `subagent_status` checks can resolve them after a restart or interrupted run.
- **Workflow visibility**: the TUI workflow view now surfaces live agent and
  phase progress, retry/failure detail, and async subagent rows.
- **Phase fallback recovery**: workflow phases can continue after a failure
  when configured with `fallback: "continue"`, so later phases still run while
  the failed phase remains visible.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm
package layout, or install commands.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.36 sh
```
