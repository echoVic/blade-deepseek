# Orca v0.1.128

Patch release: continue the Codex-style server boundary split by moving
server pending-permission request state out of `server.rs` and into a focused
`server/permission_manager.rs` module.

## Changes

- **Dedicated permission manager module**: `PendingPermissionManager`,
  `PendingPermissionRequest`, `PendingCommandExecPermissionRequest`, and
  `ServerPermissionRequestHandler` now live in
  `server/permission_manager.rs`, where they own runtime permission request
  registration, command/exec retry request storage, and pending request
  removal.
- **Smaller server orchestrator**: `server.rs` keeps stdio orchestration,
  permission response event emission, session-scope grant persistence, active
  turn metadata merge, and command/exec retry dispatch while delegating pending
  permission bookkeeping to the manager.
- **Behavior-compatible permission flow**: runtime `request_permissions`
  waits, strict auto-review propagation, session-scoped filesystem and network
  grants, command/exec network retry, command/exec filesystem retry, and
  unknown-request errors keep the same server protocol shape.
- **Documented P0 slice**: added a focused implementation plan for the
  permission manager boundary so the next server lifecycle split can continue
  from a smaller `server.rs`.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, thread ids, turn ids, permission request
ids, permission response wire shapes, or command/exec retry semantics.

This release is an internal architecture slice. Server-mode turn execution,
command/exec behavior, shell sessions, workflows, subagents, history
projection, and server/TUI/headless runtime execution remain unchanged.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.128 sh
```
