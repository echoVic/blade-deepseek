# Orca v0.1.46

Patch release: structured hook action validation.

## Changes

- **Strict declared hook actions**: JSON hook stdout that includes `action`
  now rejects unsupported action names instead of treating the full JSON blob as
  injected context.
- **Required structured fields**: `{"action":"modify"}` now requires a string
  `modified_target` or `modified_input`, and `{"action":"inject"}` now requires
  a string `context`.
- **Compatibility preserved**: plain non-JSON stdout and JSON without `action`
  remain injected context, matching the existing compatibility behavior.
- **CLI regression coverage**: `orca exec --output-format jsonl` now has a
  regression test proving typoed structured hook actions surface as JSONL error
  events and failed sessions.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm package
layout, install commands, or successful hook outputs. Hook commands that emit
JSON with a typoed or malformed `action` now fail visibly instead of silently
continuing.

## Install

```bash

```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.46 sh
```
