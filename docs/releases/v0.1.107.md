# Orca v0.1.107

Patch release: surface tool-call argument streaming progress end to end, add a
streaming idle-timeout guard, and fix env-var proxy and hook-timeout handling.

## Changes

- **Tool-call progress tracking**: a new `ToolCallProgress` type and
  `tool.call.progress` event stream through the provider, runtime, and server,
  reporting how many argument bytes have been received for an in-flight tool
  call.
- **Streaming idle timeout**: SSE parsing now enforces an idle-timeout guard so
  a stalled provider stream fails fast instead of hanging.
- **TUI progress display**: the TUI renders tool-call argument progress with
  cache optimizations to keep the live viewport responsive.
- **Fixes**: environment-variable proxy configuration and hook-timeout output
  handling are corrected, and thread test tooling is refactored for more stable
  tests.

## Compatibility

Additive. The new `tool.call.progress` event and `ToolCallProgress` provider
step extend existing event/provider enums without changing prior event names,
CLI flags, permission profile shape, install commands, or npm package layout.
Clients that ignore unknown events are unaffected.

