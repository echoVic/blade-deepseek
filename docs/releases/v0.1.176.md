# Orca v0.1.176

Patch release: add server-mode `command/exec/list` for active process recovery.

## Changes

- **Command exec list**: server-mode clients can now send
  `command/exec/list` to recover the active `command/exec` process handles
  currently owned by the server.
- **Recoverable snapshots**: each listed process includes `processId`, the
  original command argv, `cwd`, `status`, `streamOutput`, `outputBytesCap`,
  `stdoutBytes`, and `stderrBytes`, giving reconnecting clients enough state to
  resume reads, writes, resizes, or termination by process id.
- **Completion cleanup**: the server keeps its existing pre-dispatch drain, so
  completed processes emit their normal `command_exec_completed` event before
  the next list response and are omitted from the active process snapshot.
- **Stable ordering**: process snapshots are sorted by `processId` for
  deterministic app-server responses.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, or existing JSONL event names.

Existing `command/exec`, `command/exec/read`, `command/exec/write`,
`command/exec/resize`, `command/exec/terminate`, `shell/list`, `shell/read`,
`shell/capabilities`, and PTY behavior is unchanged.

Example request:

```json
{
  "id": "cmd-list",
  "method": "command/exec/list",
  "params": {}
}
```

Example response:

```json
{
  "event": "command_exec_listed",
  "id": "cmd-list",
  "processes": [
    {
      "processId": "process-1",
      "command": ["sh", "-lc", "sleep 30"],
      "cwd": "/workspace",
      "status": "running",
      "streamOutput": true,
      "outputBytesCap": 4096,
      "stdoutBytes": 12,
      "stderrBytes": 0
    }
  ]
}
```



```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.176 sh
```
