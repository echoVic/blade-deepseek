# Orca v0.2.7

Patch release: share transcript thread-item projection through `orca-core`.

## Changes

- **Core-owned transcript item projection**: user messages, persisted history
  messages, assistant-message text, proposed plans, and reasoning items now use
  reusable projection types from `orca-core` instead of private runtime-local
  definitions.
- **Stable TUI/server wire shape**: runtime projection still emits the same
  JSON item shapes for live streams, active steer user messages, and
  history/read/list responses, but those paths now construct the visible
  transcript items through one typed boundary before serialization.
- **Lower transcript drift risk**: the slice narrows the path toward a
  Codex-style `ThreadItem` protocol layer while keeping behavior unchanged for
  TUI cards users already rely on: assistant messages, plans, reasoning,
  active user prompts, and resumed conversation history.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, JSONL event names, or current server/TUI
item JSON.

This release only changes ownership of typed transcript projection helpers and
adds tests that lock the existing wire shapes.
