# Orca v0.1.56

Patch release: shared tool item status and error projection helpers.

## Changes

- **Shared error normalization**: realtime app-server item writers and
  persisted thread projections now use the same `tool_item_projection` helper
  to normalize `exit_code` / `exitCode` metadata into public camelCase
  `error.exitCode` payloads.
- **Shared status check**: realtime MCP and dynamic tool item helpers now use
  the shared projection boundary for completed-status checks before emitting
  success result/content payloads.
- **Projection guardrail**: the ownership test now covers the shared error
  normalizer and completed-status helper, preventing `server_runtime` and
  `thread_store` from growing separate copies again.
- **Behavior coverage**: projection unit tests lock both snake_case and
  camelCase exit-code input normalization, plus the strict completed-status
  predicate.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release keeps the existing `mcpToolCall` / `dynamicToolCall` wire payloads and
continues reducing internal schema drift between realtime streams and persisted
thread history.
