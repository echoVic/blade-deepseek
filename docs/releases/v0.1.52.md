# Orca v0.1.52

Patch release: MCP resource capability caching.

## Changes

- **Cached MCP capabilities**: Orca now stores the MCP `initialize` result's
  server capabilities on each connected client.
- **Quieter all-server discovery**: all-server `list_mcp_resources` and
  `list_mcp_resource_templates` calls skip servers that did not declare
  `capabilities.resources`, avoiding noisy `resources/list` failures from
  tools-only MCP servers.
- **Strict server filters preserved**: specifying `server` still targets that
  selected server directly, so config mistakes or unsupported resource methods
  remain visible when a user or model asks for a specific server.
- **Stable stdio tests**: MCP stdio fixture startup timeouts in tests were
  relaxed while preserving short tool-call timeout coverage.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode wire
shapes, npm package layout, install commands, MCP tool routing, or explicit
`server` filter behavior. All-server resource/template discovery may now omit
tools-only MCP servers instead of returning unsupported-resource errors for
them.

## Install

```bash

```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.52 sh
```
