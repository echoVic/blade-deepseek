# Orca v0.1.29

Patch release: refactor TUI session preloading for clarity and add comprehensive test coverage.

## Changes

- **Refactor session preloading logic**: extracted `startup_preloaded_transcript` variable to simplify `run_tui_inner` flow control. Fixed ownership issues with history session messages and plan data cloning.
- **New `existing_goal_session_id` helper**: centralizes session ID validation for goal operations, reducing duplicated logic in the agent loop thread.
- **Test coverage**: added unit tests covering session restoration, goal control flow, and preloaded transcript edge cases.

## Verification

```bash
cargo test --workspace
cargo clippy -p orca-core -p orca-tools -p orca-runtime -p orca-tui --all-targets
```

After publishing, verify the public artifacts:

```bash
node scripts/release/verify-published.mjs --version 0.1.29 --repo echoVic/blade-deepseek --package @blade-ai/orca --bin orca
```

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.29 sh
```
