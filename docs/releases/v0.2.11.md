# Orca v0.2.11

Patch release: route TUI keyboard shortcuts through a context-aware resolver.

## Changes

- **Context-aware shortcut resolver**: global, composer, running-turn, and
  approval-dialog shortcuts now resolve through one typed TUI shortcut boundary.
- **More reliable TUI key behavior**: the same key can be tested by context, so
  `up` remains prompt history in the composer, scrolls during a running turn,
  and moves selection in approval dialogs.
- **Shortcut help alignment**: the visible shortcut help path now reads through
  the same resolver-backed hint API, making future keymap and task-control
  changes easier to verify.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, approval options, or shortcut behavior.

This release only changes the internal TUI shortcut dispatch boundary and its
tests. Existing key combinations keep the same meaning.
