# Orca v0.1.40

Patch release: workflow evidence reporting and contract validation.

## Changes

- **Workflow evidence bundles**: new `WorkflowEvidenceBundle` types and state
  storage enable exporting standardized run evidence (inputs, outputs, timing,
  status) from any workflow execution.
- **Report generation module**: renders evidence bundles into both Markdown
  (human-readable) and JSON (machine-consumable) formats via
  `workflow::report`.
- **Automatic lifecycle evidence capture**: the workflow runner now writes
  evidence entries at spawn, phase-transition, completion, and failure
  checkpoints without manual instrumentation.
- **Contract validation tests**: new test cases verify evidence bundle
  serialization/deserialization round-trips and report rendering correctness.
- **Benchmark data**: adds `docs/reports/workflow-limit-benchmark.json` and
  `.md` documenting concurrency and scale limits observed during harness runs.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm
package layout, or install commands.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.40 sh
```
