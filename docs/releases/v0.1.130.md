# Orca v0.1.130

Patch release: continue the Codex/package-3-inspired runtime boundary split by
moving async subagent worker launch and completion persistence out of
`subagent_execution.rs` and into a focused `subagent_async_worker.rs` module.

## Changes

- **Dedicated async worker module**: async subagent worker entrypoint execution,
  child-agent runtime setup, task-registry running/completed/failed writes,
  usage persistence, async result payload shaping, worker process spawn, and
  async worktree handoff now live in `subagent_async_worker.rs`.
- **Smaller subagent executor**: `subagent_execution.rs` keeps synchronous
  subagent execution, subagent batch execution, schema validation, and shared
  worktree result formatting while delegating async launch to the worker module.
- **Behavior-compatible async flow**: async launch still returns the same
  `async_launched` payload, `subagent-worker` CLI arguments keep the same shape,
  `subagent_status` reads the same persisted task records, schema failures still
  persist failed tasks, and worktree cleanup/preservation text remains shared.
- **Documented boundary**: added a TDD implementation plan and an ownership test
  so future reducer-style subagent work keeps worker spawn/completion details out
  of the parent executor.

## Compatibility

No changes to CLI flags, `subagent-worker` arguments, async launch result JSON,
task ids, task-registry storage layout, `subagent_status` output shape,
subagent schema validation behavior, worktree preservation/cleanup semantics,
sync subagent execution, batch subagent execution, provider requests, model
selection behavior, install commands, npm package layout, thread ids, turn ids,
workflows, shell sessions, or server/TUI/headless runtime execution.

This release is an internal architecture slice. Async subagent launch,
completion, schema-failure persistence, task status lookup, and sync/batch
subagent behavior remain unchanged.

## Install

```bash

```

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.130 sh
```
