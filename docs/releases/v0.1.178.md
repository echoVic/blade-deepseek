# Orca v0.1.178

Patch release: introduce a named runtime turn kernel for the next Codex-style
turn-state consolidation.

## Changes

- **RuntimeTurnKernel seed**: `RuntimeTurnKernel` now owns the per-sampling
  request state together with the runtime turn reducer.
- **Provider-owned kernel construction**: provider response handling now creates
  tool-dispatch sampling state through `RuntimeTurnKernel` before handing it to
  tool turns.
- **Reducer-ready boundary**: the new kernel exposes reducer access from the
  same object that owns sampling state, preparing the next state-transition
  slices without changing tool behavior.
- **Architecture guardrail**: a focused runtime test locks the new boundary so
  provider turns cannot allocate sampling request state directly.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, or JSONL event names.

Tool dispatch, approval overlay behavior, readonly and subagent batching, normal
tool result recording, goal progress recording, and provider response handling
are behavior-preserving.
