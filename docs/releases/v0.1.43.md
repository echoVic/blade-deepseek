# Orca v0.1.43

Patch release: runtime-owned agent turn-loop orchestration.

## Changes

- **Lifecycle-owned turn opening**: compaction, turn start, turn-start result
  folding, model routing, and active steer injection now compose through a
  runtime lifecycle step.
- **Lifecycle-owned provider cycle**: provider calls, provider result folding,
  reactive prompt-too-long compaction, assistant response recording, memory
  extraction, provider tool-request extraction, and tool-turn dispatch now sit
  behind a single provider-cycle boundary.
- **Runtime turn iteration and loop**: the agent loop now delegates each turn
  iteration and retry/return folding to lifecycle-owned steps instead of
  matching provider-cycle results directly.
- **Narrower agent-loop call surface**: runtime turn-loop inputs and child
  executors are grouped into lifecycle-owned input objects, keeping the entry
  point closer to assembly code and making the next runtime refactors easier to
  test in isolation.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm package
layout, install commands, or model/tool behavior.

## Install

```bash

```

```bash

```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.43 sh
```
