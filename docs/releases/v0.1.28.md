# Orca v0.1.28

Patch release: drop the legacy DeepSeek model identifiers and add up-front JSON Schema validation for tool arguments before any tool runs.

## Changes

- **Drop legacy `deepseek-chat` / `deepseek-reasoner`**: `validate_model` and `allowed_models()` now only accept `auto`, `deepseek-v4-flash`, and `deepseek-v4-pro`. The TUI model picker, fixture, examples, and docs all moved to the V4 model names. Configs or `--model` flags that still pass the old identifiers will fail loudly with `unsupported model 'deepseek-chat'. Allowed models: auto, deepseek-v4-flash, deepseek-v4-pro` instead of silently routing somewhere unexpected.
- **Tool argument schema validation**: `orca-tools` now validates `ToolRequest.arguments` against each tool's `input_schema` (type / required / enum / properties / items, recursively) before dispatching to the tool. The controller surfaces validation failures as `tool arguments failed schema validation: …`, so the model gets a clear, structured error in the tool result instead of a malformed call reaching `bash`, `edit`, `write_file`, etc.
- **TUI text-wrap correctness**: rewrote the width calculation used for streaming output and added unit tests so wide-character and ANSI-styled segments no longer compute the wrong wrap width on terminals near the cutoff.
- **Fixture & docs cleanup**: `deepseek_fixture.rs`, `docs/subagent-async-demo.sh`, and `docs/subagent-enhancement-plan.md` updated to the V4 model names.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.28 sh
```
