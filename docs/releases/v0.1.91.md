# Orca v0.1.91

Patch release: unify runtime permission request merging so `request_permissions`
and automatic bash network retries share the same turn overlay path.

## Changes

- **Shared permission overlay merge**: `TurnPermissionOverlay` now owns the
  permission request/response merge path for file-system write roots, network
  domain grants, and `strictAutoReview`.
- **Request permissions alignment**: the model-visible `request_permissions`
  tool now uses the shared overlay entrypoint instead of duplicating write-root
  and network-domain merge logic.
- **Bash retry alignment**: model-visible `bash` network permission retries now
  use the same overlay entrypoint before rebuilding the retry sandbox.
- **Regression coverage**: lifecycle contract tests now lock down both network
  and file-system grant merging, plus the existing request-permission and bash
  write-root behavior.

## Compatibility

No changes to tool names, JSON protocol methods, permission response shape,
TOML permission profile syntax, install commands, or npm package layout.

This release is an architecture slice toward a single Codex-style runtime
permission boundary. Public behavior is preserved while reducing duplicated
permission merge rules across tools.
