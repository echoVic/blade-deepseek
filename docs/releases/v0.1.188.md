# Orca v0.1.188

Patch release: reuse `RuntimeStepCapabilitySnapshot` at the provider-cycle
boundary.

## Changes

- **Provider-cycle capability bundle**: `RuntimeProviderCycleInput` now carries
  request-scoped runtime capability refs through `RuntimeStepCapabilitySnapshot`
  instead of repeating the same flat fields.
- **Single assembly point**: `runtime_turn_iteration` assembles the capability
  bundle once from turn input, including instructions, memory, MCP registry,
  hooks, cancellation, task registry, workflow IPC, permission handling, and
  user-input handling.
- **Step-context handoff**: `provider_turn` now builds a `RuntimeStepSnapshot`
  from that capability bundle and passes it into `RuntimeStepContext` without
  expanding capability refs again.
- **Architecture coverage**: a source-level regression test locks the
  provider-cycle boundary so future provider/tool routing work keeps the shared
  capability snapshot instead of reintroducing duplicate flat fields.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, JSONL event names, or tool behavior.

This release is an internal runtime architecture slice. Provider turn execution,
compaction, response handling, and tool dispatch keep the same behavior while
the provider-cycle handoff becomes smaller and better aligned with the
request-scoped step context.
