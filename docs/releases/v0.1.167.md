# Orca v0.1.167

Patch release: introduce a runtime-owned sampling request state for tool-turn
permission grants.

## Changes

- **Sampling request state seed**: `RuntimeSamplingRequestState` now lives in
  `step_context.rs` as the first explicit per-sampling request-state object.
- **Permission overlay ownership**: the sampling state owns the
  `TurnPermissionOverlay`, and normal tool turns borrow it through
  `permission_overlay_mut()` instead of allocating a local overlay inside
  `tool_turn`.
- **Provider-cycle handoff**: provider response handling creates the sampling
  state before dispatching tool turns, keeping the state boundary near the model
  response that triggered the tool execution.
- **Regression coverage**: the runtime architecture contract now requires the
  sampling-state type, requires tool turns to receive it, and rejects production
  `TurnPermissionOverlay::default()` allocation inside `tool_turn`.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice. Provider responses, normal tool
execution, permission requests, bash sandbox grants, workflows, subagents, and
history projection keep the same user-visible behavior as v0.1.166.
