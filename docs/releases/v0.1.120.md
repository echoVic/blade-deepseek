# Orca v0.1.120

Patch release: continue the P0 runtime reducer split by moving model-route
orchestration out of `lifecycle.rs` and into a focused `runtime_model_route`
module with a grouped input boundary.

## Changes

- **Dedicated model-route module**: `RuntimeModelRouteStep` now lives in
  `runtime_model_route.rs` instead of `lifecycle.rs`.
- **Grouped route input**: `RuntimeModelRouteInput` groups the actor, model
  selection, provider config, cost tracker, events, sink, and delta flag so the
  extracted boundary does not preserve the old long argument list.
- **Preserved opening composition**: `RuntimeTurnOpeningStep` still composes
  compaction, turn start, model routing, and steering in the same order, but it
  now imports model routing through the focused module.
- **Smaller lifecycle surface**: `lifecycle.rs` keeps the actor, lifecycle
  state, `RuntimeModelTurn`, and surrounding primitives while shedding another
  reducer-sized orchestration slice.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, turn-start
events, model routing, cost model tracking, compaction, steering, tool
execution, approvals, workflows, shell sessions, and history persistence remain
unchanged.



```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.120 sh
```
