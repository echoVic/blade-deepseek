# Orca v0.2.29

Patch release: extend runtime ownership into the host control plane, finalize
structured mention bindings, and polish the TUI interaction surface.

## Changes

- **Runtime host control plane**: a process-owned `RuntimeHost` and bounded
  `ThreadActor` now own conversation state and hosted operations. Typed
  operation handles and completion terminals make headless and TUI lifecycle
  ownership explicit, with bounded mailboxes and joined shutdown.
- **Structured mention bindings**: `@` mentions resolve exact files, skills,
  plugins, and MCP resources through typed bindings instead of free-form path
  text. Search results, submission preparation, and history preserve those
  bindings and recover rejected or stale submissions with a user-facing error.
- **Cancellation ownership**: hosted headless sessions and TUI turns use the
  same operation boundary, so cancellation and completion state cannot leak
  into the next operation.
- **TUI interaction polish**: mouse selection and clipboard copying are handled
  explicitly, input-history tests use isolated paths, composer and welcome
  prompts explain command/file/skill mentions, and the status line formats token
  and cost metadata more clearly.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, JSONL event names, or existing shortcut
meaning.

The release keeps existing mention syntax and TUI flows while making resource
binding explicit and improving error recovery.

## Upgrade

```bash
npm install -g @blade-ai/orca@0.2.29
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.29 sh
```
