# Orca v0.2.25 (Candidate)

Feature release: make permission-profile network proxy connections bounded,
cancellable, and owned through TUI/server command shutdown.

Status: pre-publish candidate. Focused lifecycle and cross-surface tests pass.
The complete workspace, site, release-script, live-provider, remote Actions,
GitHub Release, npm, and public-install gates remain required before this note
can be marked released.

## Why

TUI bash and server `command/exec` can run behind Orca's managed network policy
proxy. The previous listener created one detached OS thread for every accepted
connection. Dropping the proxy stopped only the listener, so a stalled request
or CONNECT tunnel could outlive the command and TUI turn that owned it. There
was also no connection ceiling, request/header lines were unbounded before
parsing, and permission-block reports accumulated in unbounded channels.

## Changes

- **One supervisor owner**: `RuntimeNetworkProxy` now owns one named Tokio
  supervisor thread. The supervisor owns the listener, Hickory DNS resolver,
  connection semaphore, shutdown signal, and every connection task in one
  `JoinSet`; no per-connection thread is detached.
- **Bounded connection admission**: at most 32 connections run concurrently.
  Excess clients receive `503 Service Unavailable` with
  `x-proxy-error: connection-limit` without spawning work.
- **Bounded framing and reporting**: request lines, individual headers,
  aggregate headers, and header count are checked before parsing. Oversized
  framing receives a bounded `431` response. Network-block reports use a shared
  eight-record channel and non-blocking send, so a command that repeatedly hits
  policy cannot grow an unbounded retry queue or stall a proxy worker.
- **Cancellable transport lifecycle**: DNS lookup, upstream connect, and socket
  reads/writes have explicit deadlines. Proxy shutdown stops admission, aborts
  and awaits every active connection task, closes both sides of CONNECT
  tunnels, and joins the supervisor thread before returning.
- **Cross-surface ownership**: runtime bash, the TUI bash tool path, and server
  `command/exec` all construct the same bounded block-report channel and retain
  the proxy handle for exactly the lifetime of the owned command.

## Compatibility

CLI flags, permission-profile TOML, TUI interactions, server methods and JSONL
events, proxy environment variables, block-report content, task persistence,
and provider behavior remain unchanged. Overload and malformed/oversized proxy
framing now receive explicit bounded HTTP errors instead of consuming
unbounded resources.
