# Orca v0.2.20

Patch release: keep the terminal UI readable and actionable when prompts,
goals, plans, commands, approval previews, and session metadata are long.

## Changes

- **Large paste placeholders**: bracketed pastes over 1,000 characters display
  as `[Pasted Content N chars]` while the complete text is retained and expanded
  for slash handling, history, user-input responses, and model submission.
- **Bounded user transcript**: submitted paste placeholders remain compact in
  the live conversation; resumed or otherwise long user messages render as a
  three-row preview with their original character count.
- **Goal compaction**: the persistent Goal banner reserves room for status,
  elapsed time, tokens, and auto-continuation. Goal transcript notices use an
  80-column objective summary and unchanged automatic notices are deduplicated.
- **Plan and tool rows**: Task Plan steps and archived checklist rows use
  display-width-aware ellipses. Tool targets reserve space for the terminal
  status so `(completed)`, `(failed)`, and running state remain visible.
- **Approval safety**: long targets, preview rows, and option labels are bounded
  before rendering, preventing wrapped content from displacing the approval
  decisions below the popup.
- **Navigable menus**: slash-command and file-mention menus now render a moving
  12-row window around the selected item.
- **Responsive status line**: permission mode and context pressure have priority
  on narrow terminals; token, cost, copy, and shortcut metadata appear only
  when space remains.
- **Permission colors**: `suggest` uses blue, `auto-edit` violet, `full-auto`
  red, and `plan` teal across Dark, Light, Solarized, and Catppuccin themes.

## Compatibility

No changes to CLI flags, slash-command syntax, provider or model configuration,
approval policy semantics, tool execution, Goal persistence, history files,
server JSONL events, MCP configuration, hooks, workflows, or npm package layout.
The release changes TUI presentation only; full prompt and approval payloads
remain intact.

## Verification

pastes, placeholder deletion, complete submission expansion, long approval
content, 20-item menus, narrow Task Plan and tool rows, responsive status-line
