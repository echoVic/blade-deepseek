# Orca v0.1.112

Patch release: group normal tool-turn inputs behind
`RuntimeNormalToolTurnContext` while preserving CLI, TUI, server, tool,
approval, workflow, and history behavior.

## Changes

- **Grouped normal tool-turn context**: `run_normal_tool_turn` now receives one
  `RuntimeNormalToolTurnContext` instead of a long normal-tool argument list.
- **Dispatch cleanup**: `run_tool_turns` constructs that context for the normal
  tool branch, keeping cursoring, readonly batching, and terminal outcome
  folding unchanged.
- **Behavior compatibility**: normal tool execution, approval, result
  recording, plan-state recording, permission overlays, workflow/background
  state, and child executor handoff keep the same runtime behavior.
- **Architecture guardrails**: runtime ownership tests now reject regressions
  that re-expand normal tool-turn execution into the old long call list.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, approval prompts, or model-visible tool behavior.

This release continues the Codex/package-3-inspired tool-system convergence
work by shrinking the normal tool-turn call surface after grouping the approval
gate context.
