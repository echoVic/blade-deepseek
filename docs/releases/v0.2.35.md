# Orca v0.2.35

Feature release: interrupt sequential normal tools through one runtime-owned
lifecycle and return the TUI to a clean next prompt only after cleanup.

## Why This Release

Sequential bash, external-tool, and MCP execution previously passed through a
borrowed normal-tool context and an inline router path. The handler could see a
cancel token, but admission, worker lifetime, panic handling, permission state,
terminal selection, and cleanup ownership were split across temporary actors.
An interrupt or callback failure therefore lacked one boundary that could prove
the worker had stopped before the turn settled or the next prompt began.

This was a lifecycle and state-ownership defect, not a missing cancellation
check. Patching individual handlers would have left the same duplicate owners
and divergent permission state in place.

## Changes

- **One normal invocation owner**: `RuntimeToolCallRuntime` owns admission,
  execution-start state, the child cancel token, worker, join, panic
  classification, permission delta, and exactly-once terminal.
- **Reliable interrupt and recovery**: `CooperativeCancel` signals already
  started work and waits for process, proxy, or transport cleanup before the
  RuntimeHost operation ends or another TUI prompt is admitted.
- **Truthful mutation semantics**: `WaitForTerminal` stops later sibling
  admission without cancelling the active handler and preserves its observed
  result. Natural completion still wins a cancellation race.
- **Panic isolation**: a worker panic after start becomes an indeterminate tool
  terminal. Output, permission, and MCP callback panics cannot detach the
  worker during unwind.
- **Typed bounded interaction bridges**: output chunks, permission requests,
  and MCP elicitation cross fixed-capacity typed channels while borrowed TUI and
  server handlers stay with the parent turn owner.
- **Canonical permission state**: an invocation returns a typed permission
  overlay delta, which is merged before the next sibling call takes its
  snapshot.
- **Observed results stay authoritative**: output publication failure remains a
  separate I/O error and cannot replace an already observed tool result or make
  the tool retry-safe.
- **Old path deleted**: the borrowed normal execution context, fallback executor
  owner, direct inline router path, and source-shape ownership tests are gone.

## Compatibility

CLI arguments, TUI keys and flows, server methods and JSONL wrappers, persisted
records, tool identities and schemas, approval and permission payloads, MCP
elicitation payloads, shell task ids, and DeepSeek request behavior remain
compatible. Runtime-special tools, workflows, and subagents keep their existing
owners; subagent lifecycle remains the explicit P1.2c follow-up.

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.35
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.35 sh
```
