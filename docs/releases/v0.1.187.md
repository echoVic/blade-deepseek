# Orca v0.1.187

Patch release: group request-scoped runtime capability refs behind
`RuntimeStepCapabilitySnapshot`.

## Changes

- **Step capability snapshot**: `RuntimeStepSnapshot` now carries
  `RuntimeStepCapabilitySnapshot` as the named bundle for request-scoped runtime
  capabilities.
- **Narrower step surface**: instructions, memory, MCP registry, hooks,
  cancellation, task registry, workflow IPC, permission handling, and
  user-input handling no longer live as flat fields on `RuntimeStepSnapshot`.
- **Tool dispatch routing**: `tool_turn` now reads the capability bundle through
  `RuntimeStepSnapshot::capabilities()` before dispatching child-tool checks,
  subagent batches, readonly batches, and normal tool turns.
- **Architecture coverage**: a source-level regression test locks the new
  bundle boundary so future step-context work does not reintroduce flat
  capability fields.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, JSONL event names, or tool behavior.

This release is an internal runtime architecture slice. Provider response
handling and tool dispatch keep the same execution behavior while the
request-scoped capability API becomes smaller and easier to extend.
