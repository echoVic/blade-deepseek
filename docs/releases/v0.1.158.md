# Orca v0.1.158

Patch release: make permission reduction consistently instance-owned by
`RuntimeTurnReducer`.

## Changes

- **Instance-owned permission reducer**: removed the static
  `RuntimeTurnReducer::permission()` accessor so permission overlay mutation now
  flows through reducer instances like runtime directives and completed-tool
  goal progress.
- **Turn/thread extension plumbing**: normal tool execution and bash invocation
  contexts can now carry turn/thread extension stores to construct the reducer
  at the point permission state is reduced.
- **Runtime actor compatibility**: direct `RuntimeToolActorContext` callers get
  actor-owned extension stores as a compatibility fallback, while the normal
  agent turn path still passes the live turn/thread stores from
  `ToolExecutionContext`.
- **Behavior preservation**: model-visible `request_permissions`, bash network
  and filesystem auto-escalation, and router overlay transfer keep the same
  permission request / retry / merge behavior.
- **Regression coverage**: architecture tests now reject the old static
  permission reducer accessor and require runtime-special, runtime-bash, and
  router call sites to create reducer instances.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice in the Codex/package-3 direction:
runtime-owned reducers now consistently handle completed-tool goal progress,
runtime directives, and permission overlay mutation through instance state.
