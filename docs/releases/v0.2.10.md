# Orca v0.2.10

Patch release: make TUI context-compaction notices explain why compaction happened.

## Changes

- **Compaction reason in the TUI**: runtime `context.compacted` events now keep
  their reason, strategy, collapsed-message count, and status text when they are
  projected into the TUI.
- **Clearer long-session recovery**: DeepSeek TUI sessions now distinguish
  context compaction near the token limit, at the hard limit, and after
  prompt-too-long recovery instead of showing only a generic before/after
  message total.
- **Manual compact fallback**: `/compact` still works through the existing TUI
  path and now fills a stable `manual` compaction reason so the display path is
  consistent with runtime-owned automatic compaction notices.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, approval prompts, task panel actions, or
compaction policy.

This release only changes the TUI event payload and display text for compaction
notices. Runtime JSONL event names and provider behavior are unchanged.
