# Orca v0.1.10

Tool-system alignment and documentation release.

## Changes

- Move built-in tool handling onto a canonical registry with capability metadata, aliases, renderer hints, and spec-derived approval behavior.
- Add `glob` as the preferred file discovery tool while keeping `list_files` as a compatibility alias for older prompts and sessions.
- Keep missing-path and empty-search read results non-fatal: `glob`, `list_files`, and `grep` now report empty output semantics without marking the tool call failed.
- Route built-in, MCP, and external tools through the same registry surface so model-visible schemas and runtime dispatch stay aligned.
- Hide context-scoped tools unless they are actually usable, including `update_goal` outside persistent goal turns.
- Preserve nested-subagent depth rejection semantics so over-depth subagent calls fail directly instead of being converted into approval prompts.
- Update README, harness contract, tool comparison docs, and the website to reflect the `v0.1.10` tool surface.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```
