# Orca v0.2.17

Patch release: keep persistent Goal activity time cumulative across automatic
continuations and restore Goal sessions without resetting their usage or
identity metadata.

## Changes

- **Cumulative Goal activity**: the active Goal timer adds the persisted time
  from completed Goal turns to the current turn's live elapsed time, so the
  display no longer restarts after each automatic continuation.
- **Ordinary turn parity**: non-Goal turns and Goals that are paused or in a
  terminal status keep the existing current-turn-only activity timer.
- **Metadata-preserving resume**: `/goal resume` keeps the objective, token
  budget, tokens used, elapsed time, and original `created_at` value.
- **Atomic restoration**: same-session reactivation updates the existing
  record in place. Cross-session restoration writes the complete target record
  and pauses the source in one persisted Goal-store replacement.
- **Collision guard**: cross-session restoration refuses to overwrite an
  existing target Goal and leaves both records unchanged.
- **Failure-safe TUI state**: history mode, preloaded history, and the live TUI
  session are committed only after Goal persistence succeeds.
- **Restored projection ordering**: a restored Goal is projected before the
  first `TurnStarted` event, so the first running frame includes the persisted
  elapsed-time base.

## User-visible Timing

For an active Goal, Orca now renders:

```text
displayed time = persisted completed-turn time + current-turn elapsed time
```

Approval and user-input waits that occur inside the current turn count toward
that turn's elapsed time. Time between automatic continuations, while the Goal
is paused or terminal, and while Orca is closed is excluded.

The current-turn delta is persisted when the turn returns. A process crash can
therefore lose the unfinished current-turn delta, but it does not reset time
already persisted from earlier turns.

## Resume And Persistence Safety

Resume reads the full source record and changes only its active/session
placement and `updated_at`. Same-session resume preserves the existing session
identity. If history restoration uses another session id, Orca creates the
active target and pauses the source from the same in-memory database snapshot,
then saves them through one temporary-file replacement.

A missing source, occupied target, or store write failure does not replace an
existing Goal. The TUI also retains its prior history configuration, preloaded
transcript, and live session when restoration cannot be persisted.

## Compatibility

No changes to CLI flags, slash-command syntax, provider/model configuration,
MCP configuration, hooks, workflows, approvals, shortcuts, or npm package
layout.

The `goals_1.json` record shape is unchanged. Existing Goal files load in
place; no JSON migration or manual data conversion is required.

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.17
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.17 sh
```
