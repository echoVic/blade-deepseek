# Orca v0.1.170

Patch release: move normal tool result recording into the runtime-owned sampling
request state.

## Changes

- **Result-recording ownership**: `RuntimeSamplingRequestState` now records
  normal tool results through the same request-state boundary that owns the
  permission overlay, tool cursor, and dispatch windows.
- **Terminal folding cleanup**: approval-required results and failed synchronous
  subagent results are folded inside the sampling request state instead of a
  `tool_turn` helper.
- **Normal tool context cleanup**: `RuntimeNormalToolTurnContext` now receives
  the sampling request state directly and borrows its permission overlay during
  tool execution.
- **Regression coverage**: architecture contracts now require normal result
  recording to live in the sampling state, and behavior tests cover
  approval-required plus subagent-failure terminal outcomes after the result is
  recorded.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice. Provider responses, normal tool
execution, permission requests, readonly batching, subagent batching, workflows,
history projection, and public release packaging keep the same user-visible
behavior as v0.1.169.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.170 sh
```
