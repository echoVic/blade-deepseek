# Orca v0.1.61

Patch release: shared lifecycle item projection for realtime server streams
plus release-gate test hardening.

## Changes

- **Shared lifecycle items**: realtime app-server `fileChange` and `workflow`
  item lifecycle events now use `tool_item_projection` builders instead of
  hand-written JSON shapes in `server_runtime`.
- **Schema drift guardrail**: ownership tests now prevent `server_runtime` and
  `thread_store` from reintroducing separate fileChange/workflow item builder
  implementations.
- **Projection coverage**: unit tests lock started/completed fileChange shapes
  plus started/completed workflow shapes, including null path handling, result,
  error, task id, workflow name, and task metadata.
- **Compatibility coverage**: server-mode contract tests still verify
  fileChange and workflow item lifecycle events end to end, preserving existing
  app-server wire fields.
- **Release gate hardening**: the tag-driven Release workflow now runs the
  Rust test harness with one test thread and backtraces enabled, avoiding
  intermittent server-contract process interference on Linux release runners
  while keeping the same full `cargo test` coverage.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release is an internal projection-boundary refactor for realtime `fileChange`
and `workflow` lifecycle items.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.61 sh
```
