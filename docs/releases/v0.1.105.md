# Orca v0.1.105

Patch release: move normal tool execution behind a focused runtime executor
boundary while preserving CLI, TUI, server, workflow, permission, and
model-visible tool behavior.

## Changes

- **Runtime normal tool executor**: `RuntimeNormalToolExecutor` now owns normal
  model-visible tool execution for the runtime.
- **Shell-session bash branch**: model-visible bash shell-session execution is
  delegated through the new executor instead of being owned directly by
  `lifecycle.rs`.
- **Fallback execution path**: MCP, TOML external, and built-in normal tool
  fallback execution now flows through the same focused boundary.
- **Architecture guardrails**: the runtime lifecycle contract now checks that
  `runtime_normal_tool` owns the shell-session branch and fallback executor
  call, and that lifecycle delegates through the new executor.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or model-visible tool behavior.

This release is an internal architecture slice inspired by Codex-style request
processor boundaries and package-3-style grouped runtime context objects. It
preserves behavior while continuing to shrink `lifecycle.rs` into clearer,
reviewable runtime execution boundaries.
