# Orca v0.1.144

Patch release: isolate child-agent public entry points so `agent_child.rs`
continues shrinking into a compatibility facade while runtime-owned child-agent
modules keep the execution behavior.

## Changes

- **Focused entrypoint module**: `child_agent_entrypoints.rs` now owns
  `run_child_agent`, `run_child_agent_with_executor`, and
  `run_child_agent_prompt_with_tool_executor`.
- **Smaller facade**: `agent_child.rs` now re-exports entry points, loop setup,
  provider turn, response folding, loop runner, and shared child-agent types
  without carrying entrypoint implementation bodies.
- **Ownership coverage**: a new architecture test locks the boundary so
  child-agent entrypoint wrappers do not drift back into the compatibility
  facade.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, provider turn dispatch, provider hook execution, child compaction,
prompt-too-long recovery, response folding, tool-result folding, cost tracking,
approval policy semantics, MCP routing, external tool routing, shell session
protocol, command/exec wire shape, conversation storage, history storage, npm
package layout, or install commands.

This release is an internal ownership slice. Existing
`orca_runtime::agent_child` imports remain available through re-exports while
the implementation of child-agent model override setup, child cost tracker
creation, executor error folding, and prompt-to-request construction moves to
the focused entrypoint module.



```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.144 sh
```
