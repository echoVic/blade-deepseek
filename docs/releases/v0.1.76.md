# Orca v0.1.76

Patch release: the runtime protocol boundary is now split into focused modules while preserving the existing server-mode wire format and public protocol API.

## Changes

- **Protocol facade**: `protocol.rs` now declares and re-exports focused protocol modules instead of owning the full JSON decode and event serialization surface directly.
- **Command execution boundary**: command exec options, sandbox policy, network access, env overrides, and command argument helpers now live in `protocol/command_exec.rs`.
- **Permission boundary**: permission response decisions, grant scopes, request permission profiles, filesystem entries, and network permission request types now live in `protocol/permissions.rs`.
- **Event boundary**: `ServerEvent`, event envelopes, runtime-event mapping, legacy flat JSON encoding, and event writing now live in `protocol/events.rs`.
- **Wire parsing boundary**: `Submission`, `ClientOp`, JSON-RPC decode, package-3 permission-update parsing, and app-server approval-policy compatibility stay in `protocol/wire.rs`.
- **Focused helpers**: shell terminal parsing and shell quoting live in `protocol/shell.rs`; thread list/search parsing lives in `protocol/thread.rs`; turn input extraction lives in `protocol/turn.rs`.
- **Architecture guard**: a unit test now prevents the protocol boundary from regressing into another monolithic file.

## Compatibility

No changes to CLI flags, server-mode method names, JSON request/response shapes, realtime event names, npm package layout, direct binary install commands, thread storage formats, or TUI behavior.

This release is an architecture slice only. It keeps `crate::protocol::*` imports working while making the next server dispatch and query-kernel refactors easier to review.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.76 sh
```
