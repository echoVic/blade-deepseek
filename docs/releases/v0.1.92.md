# Orca v0.1.92

Patch release: move server-mode operation dispatch behind a focused router
boundary as the first request-processor architecture slice.

## Changes

- **Server router boundary**: `server.rs` now keeps stdio line handling,
  submission decoding, active process draining, and lifecycle cleanup, while
  decoded `ClientOp` dispatch lives in `server/router.rs`.
- **Architecture guardrail**: a new runtime architecture test locks the
  boundary so the server entry point cannot silently grow the operation match
  again.
- **Behavior preservation**: every existing server-mode operation still routes
  to the same handler functions, including submit, thread lifecycle, turn
  controls, permission responses, shell sessions, and command/exec.
- **Next refactor path**: the router boundary is the first step toward
  Codex-style per-operation request processors without changing public wire
  shapes.

## Compatibility

No changes to CLI flags, JSON protocol methods, server event names, permission
profile TOML shape, TUI behavior, install commands, or npm package layout.

This release is an internal architecture slice. It preserves current behavior
while making the next server processor extraction safer and easier to test.
