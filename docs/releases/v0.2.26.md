# Orca v0.2.26

Feature release: give TUI runtime events and user actions typed, bounded
mailboxes with explicit backpressure and receiver lifetime.

Status: released on 2026-07-15. Focused mailbox, observer, compaction, and
background-continuation tests plus complete workspace, Clippy, site,
release-script, and live-provider gates passed before publication. Release
workflow `29388934712` passed the complete test, four-platform build, GitHub
Release, npm publish, and npm release-asset jobs. The post-publish verifier
confirmed the public GitHub Release, `@blade-ai/orca@0.2.26`, and `npm exec`
installation.

## Why

The production TUI created its runtime-event and user-action lanes with
unbounded channels while the renderer deliberately drained only a bounded
batch per scheduler iteration. A long DeepSeek stream, tool output flood, or
paused terminal could therefore grow queued memory without a ceiling.

Runtime compaction and approved background continuation also serialized typed
`EventEnvelope` values as JSONL into a TUI-local writer. That writer retained
an unbounded partial frame, parsed it into a weaker local envelope, then rebuilt
the runtime event only to project it into `TuiEvent`. The serialization buffer,
rather than the typed runtime value, had become an unnecessary second source of
truth.

## Changes

- **Bounded TUI admission**: runtime events use a 256-value mailbox and user
  actions use a 64-value mailbox. Admission blocks at capacity; no admitted
  assistant, approval, error, control, or terminal value is silently dropped.
- **Typed runtime projection**: `EventSink` accepts an optional owned
  `EventObserver`. The observer receives the exact run id, sequence, timestamp,
  event type, and payload written by the sink, and observer failure is returned
  to the operation.
- **One runtime-to-TUI path**: compaction and approved background continuation
  install `TuiRuntimeEventObserver` through their runtime request boundaries.
  TUI projection consumes the original typed envelope without parsing JSON.
- **Bounded internal lanes**: provider streaming has an explicit 256-value
  channel, mention catalog refresh has an eight-value channel, and silent child
  agents use the production bounded mailboxes plus a drain worker that is
  joined before return.
- **Old path deleted**: `TuiRuntimeEventWriter`, `TuiRuntimeEventEnvelope`, the
  partial JSONL frame buffer, its deserializer, and its source-shape test no
  longer exist. Behavior tests cover capacity, blocking, wakeup, disconnection,
  complete slow-consumer output, terminal delivery, and typed projection.

## Compatibility

CLI arguments, TUI keys and interaction flows, server methods and JSONL events,
persisted records, permission handling, tool and workflow output, and DeepSeek
streaming, retry, compaction, and usage behavior remain unchanged. The intended
behavior change is blocking producer backpressure after 256 unconsumed TUI
events or 64 unconsumed user actions.
