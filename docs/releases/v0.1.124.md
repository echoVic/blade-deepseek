# Orca v0.1.124

Patch release: continue the P0 runtime reducer split by moving task/turn
lifecycle state out of `lifecycle.rs` and into a focused
`runtime_lifecycle` module.

## Changes

- **Dedicated lifecycle state module**: `RuntimeSessionLifecycle`,
  `RuntimeTaskLifecycle`, `RuntimeTaskKind`, `RuntimeTaskStatus`,
  `RuntimeTurnLifecycle`, `RuntimeTurnRunner`, `RuntimeStartedTurn`, and
  `RuntimeAdvancedTurn` now live in `runtime_lifecycle.rs`.
- **Preserved public imports**: existing callers can continue importing these
  types through `orca_runtime::lifecycle::*`; `lifecycle.rs` re-exports the
  moved state machine while keeping actor/tool/hook behavior for follow-up
  slices.
- **Behavior-compatible task metadata**: task ids, turn numbers, run-status to
  lifecycle-status mapping, shell task payloads, and `turn.started` task
  metadata are unchanged.
- **Documented P0 slice**: added a focused implementation plan for the
  lifecycle-state boundary so the next actor/context extraction can continue
  from a smaller surface.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, task ids, turn numbering, or approval
response wire shapes.

This release is an internal architecture slice. Runtime task lifecycle events,
tool events, shell task metadata, workflows, subagents, session persistence,
history projection, and server/TUI/headless turn execution remain unchanged.
