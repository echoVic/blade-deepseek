# Orca v0.1.73

Patch release: permission-profile filesystem globs now support configurable scan depth for command sandbox materialization.

## Changes

- **Configurable glob scan depth**: `[permission_profiles.<name>.filesystem]` now accepts `glob_scan_max_depth` and the protocol-aligned alias `globScanMaxDepth`.
- **Profile-chain semantics**: child profiles inherit the nearest parent `glob_scan_max_depth` unless they set their own value.
- **Bounded sandbox startup**: command sandbox glob expansion still defaults to depth `32`, clamps configured values to a safe range, and rejects globs without a static parent directory before scanning.

Example:

```toml
[permission_profiles.docs]
extends = ":read-only"

[permission_profiles.docs.filesystem]
glob_scan_max_depth = 4
"/workspace/docs/**/*.md" = "read-write"
```

## Compatibility

No changes to CLI flags, server-mode method names, npm package layout, install commands, JSONL event names, public realtime event names, or TUI display behavior.

Existing permission-profile filesystem entries keep the default scan depth and matching behavior unless `glob_scan_max_depth` / `globScanMaxDepth` is configured.

```bash

```


