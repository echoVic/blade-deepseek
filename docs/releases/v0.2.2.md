# Orca v0.2.2

Patch release: harden DeepSeek tool-call compatibility and validation. The
`update_goal` and `update_plan` tools now normalize the argument shapes DeepSeek
tends to emit, tool JSON Schemas gain nullable/`anyOf` support, and the system
prompt no longer inlines full tool schemas.

## Changes

- **Goal/plan argument normalization**: `update_goal` and `update_plan` accept
  DeepSeek's status aliases and boolean status flags (`{"complete": true}`,
  `{"completed": true}`, `{"blocked": true}`) and normalize them into the
  canonical `status` field before validation.
- **Tool schema hardening**: the `glob` and `update_goal` JSON Schemas support
  nullable parameters and `anyOf` validation, and tool validation errors now
  list the allowed and required properties to make failures self-explanatory.
- **Leaner system prompt**: the system prompt no longer inlines complete tool
  schemas and instead shows concise usage examples, keeping prompts prefix-cache
  friendly.
- **DeepSeek interaction reliability**: adds reasoning-content replay, a
  tool-count upper bound, and an empty-response retry so DeepSeek turns stay
  well-formed. `deepseek-chat` is no longer wrongly rejected by model
  validation.
- **Plan freshness reminder**: a valid plan left un-updated for a long time now
  triggers a reminder, and the TUI shows a stale-plan hint when a plan update
  fails.
- **Changelog page SEO**: the SEO check now validates the React source instead
  of rendered HTML, the changelog page drops its top global navigation link, and
  a guard prevents that navigation from being reintroduced.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire event names, or JSONL event names.

This release only improves DeepSeek tool-call robustness and validation
messaging; the approve/deny, goal, and plan semantics stay the same.

```bash
CARGO_TARGET_DIR=/tmp/blade-deepseek-target cargo test -p orca-tools --lib
npm --prefix site run build

node scripts/release/test-stage-npm.mjs
```

```bash
node scripts/release/verify-published.mjs --version 0.2.2 --repo echoVic/blade-deepseek --package @blade-ai/orca --bin orca
```

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.2
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.2 sh
```
