# Orca v0.2.13

Patch release: bound runtime task-output retention for long-running TUI bash and server command/exec sessions.

## Changes

- **Bounded task-output store**: shell and command/exec output now flows through
  a shared `TaskOutputStore` with a retention cap instead of unbounded
  per-process stdout/stderr buffers.
- **UTF-8-safe incremental reads**: retained output trimming and delta reads
  advance to valid UTF-8 boundaries, so multibyte text is not split mid-codepoint.
- **Streaming cap stability**: server command/exec streaming keeps cumulative
  stdout/stderr cap accounting even after retained output has been trimmed and
  rebased to the retained tail.
- **Visible omitted-output notices**: command/exec deltas report when earlier
  retained output was omitted, so users can tell why a stream resumes from a
  later tail.
- **Terminal output cleanup**: completed, stopped, or permission-denied terminal
  paths remove retained process output once the runtime observes the terminal
  state.

## TUI impact

Long TUI sessions can run noisy shell commands without the runtime keeping an
unbounded copy of every emitted byte. Users still get incremental command
visibility, completion output, and cap warnings, but the task-control path now
has a bounded output-retention floor for future shell, subagent, and server-mode
work.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, approval options, or shortcut behavior.

The JSON event names stay the same. Command/exec output deltas may now include a
short omitted-output notice when the runtime had to trim earlier retained bytes.

Install or pin the release with:

```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.13 sh
```
