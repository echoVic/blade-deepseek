# Orca v0.1.62

Patch release: shared text lifecycle item projection for realtime server
streams.

## Changes

- **Shared text items**: realtime app-server `agent_message`, `plan`, and
  `reasoning` item lifecycle events now use `tool_item_projection` builders
  instead of hand-written JSON shapes in `server_runtime`.
- **Protocol-boundary cleanup**: `ServerRequestWriter` now keeps text item
  state and streaming deltas, while shared builders own the final app-server
  item schema for started/completed text lifecycle events.
- **Projection coverage**: unit tests lock agent-message, plan, and reasoning
  item shapes, including the reasoning `summary` plus empty `content` field
  expected by existing clients.
- **Compatibility coverage**: server writer tests still verify assistant
  message, proposed-plan extraction, split proposed-plan tags, incomplete plan
  fallback, and reasoning lifecycle streams end to end.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release is an internal projection-boundary refactor for realtime
`agent_message`, `plan`, and `reasoning` lifecycle items.

## Install

```bash

```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.62 sh
```
