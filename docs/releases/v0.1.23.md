# Orca v0.1.23

Release verification reliability release.

## Changes

- Add retry/backoff support to `scripts/release/verify-published.mjs` so post-publish verification waits for GitHub Release, npm registry, and `npm exec --package @blade-ai/orca@<version> -- orca --version` visibility instead of failing during npm propagation.
- Extend `scripts/release/test-verify-published.mjs` with a transient `npm exec` failure case that must retry before passing, while still failing when a release is actually missing.
- Keep release metadata aligned across Cargo, README, roadmap, website metadata, tools comparison, and release notes.

## Verification

```bash
cargo test
npm --prefix site run build
npm --prefix site run check:seo
node scripts/release/test-stage-npm.mjs
node scripts/release/test-verify-published.mjs
git diff --check
```

After publishing, verify the public artifacts:

```bash
node scripts/release/verify-published.mjs --version 0.1.23 --repo echoVic/blade-deepseek --package @blade-ai/orca --bin orca
```

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```
