# Orca v0.2.8

Patch release: move command/exec sandbox and permission-profile resolution behind
a focused server module.

## Changes

- **Focused command/exec sandbox boundary**: command/exec sandbox-policy mapping,
  active permission-profile inheritance, custom filesystem roots, glob
  expansion, network domain policy, and Unix socket allowlist resolution now live
  in `server/command_exec_sandbox.rs` instead of the generic server loop.
- **Stable TUI/server behavior**: TUI bash and server `command/exec` still share
  the same `bash_sandbox_for_cwd` / command sandbox behavior and emit the same
  JSON wire shapes; the slice only changes ownership of the resolver.
- **Lower permission-regression risk**: future DeepSeek/TUI work around network
  prompts, filesystem retry approval, and task-control visibility can test the
  command sandbox boundary directly without threading more policy logic through
  the large server module.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, JSONL event names, command/exec event
payloads, or current TUI approval behavior.
