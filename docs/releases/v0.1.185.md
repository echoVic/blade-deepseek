# Orca v0.1.185

Patch release: group turn-scoped interaction handlers behind a named runtime
state boundary.

## Changes

- **Turn interaction state**: `RuntimeTurnInteractionState` is now the named
  home for turn-scoped interaction handlers. The first handler carried through
  this boundary is the permission-request handler used by `request_permissions`
  and sandbox escalation paths.
- **Narrower agent-loop context**: `AgentLoopContext` no longer exposes the
  permission handler as a parallel top-level field. `with_permission_handler`
  remains the construction API, but it now writes into the grouped turn
  interaction state.
- **Loop handoff alignment**: `runtime_turn_loop` and
  `runtime_turn_iteration` pass the grouped interaction state forward, and the
  provider/tool cycle reads the permission handler from that state at the point
  where existing dispatch still needs it.
- **Architecture guardrail**: focused contract and unit tests keep
  turn-scoped interaction handlers from drifting back into standalone
  `AgentLoopContext` fields.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, or JSONL event names.

Approval prompts, `request_permissions`, permission overlays, sandbox
escalation, tool dispatch, provider response handling, workflow tools,
subagents, and TUI approval behavior remain behavior preserving.
