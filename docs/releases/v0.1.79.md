# Orca v0.1.79

Patch release: route headless controller turns through `RuntimeThread`, extending the Codex/package-3-style runtime thread boundary from server mode into CLI exec.

## Changes

- **Headless RuntimeThread adoption**: the headless controller now creates long-lived agent state with `RuntimeThread::start` instead of directly assembling `InteractiveSession`, `RuntimeSessionLifecycle`, task registry, hooks, memory, MCP registry, and the agent loop.
- **Shared event continuity**: headless turns can pass the outer `EventFactory` into the runtime thread path, preserving monotonic JSONL `seq` values from `session.started` through `turn.started` and `session.completed`.
- **Session hook order preserved**: headless keeps the existing SessionStart and SessionEnd hook ordering, while the shared turn executor continues to emit `session.completed` by default for server and TUI callers.
- **Runtime thread API**: `RuntimeThread` and `ThreadTurnExecutor` now expose event-factory-aware request execution for entrypoints that already own session-level event emission.
- **Architecture guard**: a new runtime boundary test prevents headless `run_inner` from directly assembling lifecycle/task state or calling `run_agent_loop` outside `RuntimeThread`.

## Compatibility

No changes to CLI flags, server-mode method names, JSON request/response shapes, realtime event names, npm package layout, direct binary install commands, thread storage formats, permission-profile semantics, resume/fork behavior, cancellation, or TUI behavior.

This release is an architecture slice only. It removes duplicated headless runtime ownership while keeping CLI JSONL sequencing, hook ordering, verifier handling, history writes, and server runtime contracts intact.

```bash
npm install -g @blade-ai/orca

```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.79 sh
```
