# Orca v0.1.145

Patch release: group child-agent prompt entrypoint inputs behind a focused
context object so the child-agent public entrypoint layer keeps shrinking
without changing runtime behavior.

## Changes

- **Grouped prompt entrypoint context**: `ChildAgentPromptContext` now carries
  the prompt, subagent type, optional model override, depth, cwd, project
  instructions, memory block, and hooks for
  `run_child_agent_prompt_with_tool_executor`.
- **Shorter entrypoint API**: `run_child_agent_prompt_with_tool_executor` now
  accepts the grouped context instead of a long prompt/subagent/cwd/instruction
  argument list.
- **Facade compatibility**: `agent_child.rs` re-exports
  `ChildAgentPromptContext` alongside the existing child-agent entrypoint
  functions so runtime and TUI callers continue using the compatibility
  facade.
- **Ownership coverage**: the child-agent entrypoint architecture test now
  locks the grouped prompt context boundary and rejects the previous long
  prompt entrypoint signature.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, provider turn dispatch, provider hook execution, child compaction,
prompt-too-long recovery, response folding, tool-result folding, cost tracking,
approval policy semantics, MCP routing, external tool routing, shell session
protocol, command/exec wire shape, conversation storage, history storage, npm
package layout, or install commands.

This release is an internal ownership and API-shape slice. Existing runtime
callers can continue importing child-agent entrypoint pieces through
`orca_runtime::agent_child`, while the prompt entrypoint no longer creates a
new long-argument surface.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.145 sh
```
