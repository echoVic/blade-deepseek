# Orca v0.1.109

Patch release: route normal tools through a grouped runtime invocation object
while preserving CLI, TUI, server, workflow, permission, shell-session, and
model-visible tool behavior.

## Changes

- **Grouped normal-tool invocation**: `RuntimeNormalToolInvocation` now carries
  the normal-tool call state from the router into lifecycle actors.
- **Smaller lifecycle call surface**: router-owned normal-tool dispatch no
  longer calls the long roots/cancel lifecycle method directly.
- **Lifecycle actor entrypoint**: `RuntimeTaskActor` and
  `RuntimeToolActorContext` expose a crate-private invocation-object entrypoint
  while preserving the existing compatibility wrappers.
- **Behavior preserved**: bash shell-session execution, MCP fallback execution,
  TOML external tool fallback execution, output truncation, shell timeout,
  cancellation, and permission-overlay handling continue through the same
  executor branches.
- **Architecture guardrails**: runtime lifecycle contract tests now reject
  router regressions back to the long roots/cancel lifecycle method.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or model-visible tool behavior.

This release continues the Codex/package-3-inspired architecture work by moving
the router-to-lifecycle normal-tool path onto the same grouped invocation shape
as the executor boundary. It keeps the current execution behavior intact while
preparing the runtime for later long-running shell, worktree automation, and
async subagent slices.


```bash

```

```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.109 sh
```
