# Orca v0.1.95

Patch release: move server shell-session dispatch into a focused server
processor module.

## Changes

- **Shell processor boundary**: `server/router.rs` now delegates decoded shell
  session operations to `server/processors/shell.rs`.
- **Covered operations**: `ShellStart`, `ShellWrite`, `ShellUpdate`,
  `ShellClose`, `ShellResize`, `ShellList`, `ShellRead`, and `ShellKill` now
  share the focused shell processor entrypoint.
- **Architecture guardrail**: a new runtime architecture test keeps shell
  dispatch details out of the generic router.
- **Behavior preservation**: shell start/write/close/read/kill, shell list and
  description updates, PTY initial size and resize, pipe resize rejection, task
  stop reaping, and shell lifecycle event behavior keep the same protocol
  semantics.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, TUI behavior, install commands, or npm package
layout.

This release continues the Codex-style request-processor refactor path. Submit,
permission, and command/exec processor extractions remain follow-up slices.
