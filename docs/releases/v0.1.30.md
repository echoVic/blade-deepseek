# Orca v0.1.30

Patch release: workflow runtime overhaul and TUI experience polish.

## Changes

- **Workflow DSL and runtime overhaul**: workflow scripts now support exported `phases` and `meta` configuration, multi-stage execution, persisted marker phases, cached agent resume behavior, and richer task/run state synchronization.
- **Cleaner workflow tool boundaries**: workflow child runs isolate tool context more carefully, avoid stale conversation/tool-call residue, and report host/runtime failures through structured workflow events.
- **TUI workflow visibility**: the TUI now shows workflow/task progress, elapsed runtime, notifications, background task state, and clearer workflow result updates while long-running work continues.
- **Approval dialog clarity**: approval options now name the concrete tool, describe exact-call allow rules more clearly, and hide exact-call allow choices for dynamic-target tools where the same arguments are unlikely to repeat.
- **Changelog and release metadata**: website changelog summaries and version metadata are aligned with the new patch baseline.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.30 sh
```
