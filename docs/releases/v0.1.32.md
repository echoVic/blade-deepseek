# Orca v0.1.32

Patch release: P1 runtime protocol boundary.

## Changes

- **Typed server submissions**: server mode now decodes incoming JSONL into
  `orca_runtime::protocol::Submission` and `ClientOp` instead of matching raw
  operation strings in the server loop.
- **Typed server events**: server responses now flow through
  `ServerEvent` variants and a typed event envelope while preserving the
  existing flat JSON wire format.
- **Runtime event adapter**: mapping from internal runtime JSONL events to
  server-facing protocol events now lives in `orca_runtime::protocol`.
- **Compatibility preserved**: existing `submit` requests and response event
  names are unchanged for current clients.
- **Architecture docs**: the production roadmap now treats P1 as complete and
  records P2 as tool-system convergence.

## Compatibility

This release does not change public server request or response JSON shapes. For
example, `{"id":1,"op":"submit","prompt":"hello"}` still yields flat events such
as `{"id":1,"event":"message_delta","text":"..."}`.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.32 sh
```
