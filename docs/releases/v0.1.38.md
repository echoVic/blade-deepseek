# Orca v0.1.38

Patch release: history/session store boundary.

## Changes

- **SessionStore boundary**: history/session persistence now flows through a dedicated `SessionStore` object instead of a pile of free helpers.
- **Runtime call-site cleanup**: `InteractiveSession` and the runtime controller now use the same store entry point for load, resume, fork, and writer setup.
- **Architecture alignment**: this keeps the JSONL format and current history commands stable while moving Orca a little closer to Codex-style thread storage.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, or npm package layout.


