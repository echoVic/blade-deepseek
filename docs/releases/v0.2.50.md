# Orca v0.2.50

Orca v0.2.50 removes the fixed 64-turn ceiling from Goal Mode. A long-running
Goal now continues until a semantic runtime condition says it should stop,
rather than until an arbitrary counter expires.

## What Changed

- `RuntimeHost` no longer rejects a Goal continuation because its persisted
  outer-turn count reached a compile-time maximum.
- Continuation admission now depends only on Goal state, prior-turn success,
  cancellation and shutdown, queued user input, pending interaction, workflow
  ownership, plan mode, generation fencing, and Goal or verifier budget.
- The old fixed-limit rejection path and its incorrect
  `Paused(NoProgress)` projection are deleted. No-progress pauses remain tied to
  three successful outer turns that repeat the same normalized model-fixable
  gap.
- `continuation_count` remains in the run ledger and semantic events for audit
  and observability. It is not a stopping condition.
- Goal, harness, README, roadmap, release, and website documentation now state
  the same runtime contract.

## Compatibility

Goal commands, SQLite schema, existing run ledgers, semantic event fields, TUI
projection, ACP behavior, and the internal JSONL protocol are unchanged. Stored
continuation counts remain readable and continue increasing across admitted
outer turns.

## Verification

- focused Goal runtime and continuation-admission tests
- Goal runtime-host integration tests
- Goal TUI tests
- complete serial workspace tests across all targets
- Rust formatting, release helper, website build, and SEO checks

## Install

```bash
npm install -g @blade-ai/orca@0.2.50
```

Or install a native release asset:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.50 sh
```
