# Orca v0.1.141

Patch release: isolate the child-agent loop runner boundary so the
`agent_child` module keeps compatibility re-exports while loop orchestration
lives in a focused module.

## Changes

- **Focused loop-runner module**: `child_agent_loop_runner.rs` now owns
  `run_child_agent_loop_with_tool_executor` and
  `run_child_agent_with_tool_executor`.
- **Compatibility re-export**: existing `orca_runtime::agent_child` imports for
  child-agent loop-runner APIs remain available via re-export.
- **Agent-child shrink**: `agent_child.rs` no longer owns the provider loop
  body that composes child loop setup, turn-budget advancement, compaction,
  model routing, provider calls, provider error handling, response folding, tool
  request extraction, tool execution, and tool-result folding.
- **Ownership coverage**: a new architecture test locks the loop-runner
  boundary so the `agent_child` facade does not silently reacquire orchestration
  details.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, provider turn dispatch, provider hook execution, child compaction,
prompt-too-long recovery, response folding, tool-result folding, cost tracking,
approval policy semantics, MCP routing, external tool routing, shell session
protocol, command/exec wire shape, conversation storage, history storage, npm
package layout, or install commands.

This release is an internal ownership slice. Child-agent behavior for subagent
tool calls, workflow child agents, async subagents, TUI child-agent delegation,
provider bootstrap, model routing, provider turns, tool-call extraction,
tool-result folding, nested child-cost tracking, and turn-budget exhaustion is
unchanged; the loop-runner boundary is smaller and easier to keep aligned with
the rest of the Codex/package-3-inspired runtime split.

## Install

```bash
npm install -g @blade-ai/orca

```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.141 sh
```
