# Orca v0.1.117

Patch release: continue the P0 runtime reducer split by moving turn-iteration
orchestration out of `lifecycle.rs` and into a focused
`runtime_turn_iteration` module.

## Changes

- **Dedicated turn-iteration module**: `RuntimeTurnIterationStep`,
  `RuntimeTurnIterationInput`, and `RuntimeTurnIterationResult` now live in
  `runtime_turn_iteration.rs` instead of `lifecycle.rs`.
- **Behavior-compatible iteration orchestration**: `runtime_turn_loop` still
  delegates through `RuntimeTurnIterationStep`, while provider-cycle execution
  still lives behind the existing `provider_turn` boundary.
- **Smaller lifecycle surface**: `lifecycle.rs` no longer owns the per-iteration
  retry/return folding layer, leaving it focused on lifecycle state, opening,
  start, model route, and actor behavior for the next reducer-style split.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, tool
execution, approvals, workflows, shell sessions, and history persistence remain
unchanged.
