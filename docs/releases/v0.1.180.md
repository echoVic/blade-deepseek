# Orca v0.1.180

Patch release: assemble provider-response input through the runtime turn kernel.

## Changes

- **Kernel-owned response input**: `RuntimeTurnKernel` now assembles
  `RuntimeProviderResponseInput` for provider response handling.
- **Narrower state exposure**: provider response handling no longer exposes
  kernel-owned sampling state or step-context binding as separate fields at the
  provider-cycle boundary.
- **Tighter turn-state handoff**: the bound `RuntimeStepContext`, sampling
  state, events/sink, conversation/history refs, cost tracker, and background
  workflow handles now cross into response handling through one kernel-owned
  input assembly path.
- **Architecture guardrail**: a focused runtime test prevents provider turns
  from bypassing the kernel-owned response-input assembly.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, or JSONL event names.

Provider response handling, tool dispatch, approval overlays, workflow tools,
subagent batching, goal progress, and history recording remain behavior
preserving.
