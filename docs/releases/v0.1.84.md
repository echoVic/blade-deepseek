# Orca v0.1.84

Patch release: carry permission-profile Unix socket allowlists into `command/exec` sandboxing on macOS.

## Changes

- **Unix socket allowlists**: configured `[permission_profiles.<name>.network.unix_sockets]` entries with `allow` now materialize into `command/exec` sandbox state instead of being rejected as unsupported.
- **macOS Seatbelt rules**: workspace-write and read-only command sandboxes now allow configured AF_UNIX socket roots through explicit `network-bind` and `network-outbound` rules without enabling broad network access.
- **Profile inheritance preserved**: configured permission profile chains still resolve before command execution; only Unix socket entries resolved to `allow` are passed to the sandbox.
- **Cross-platform compatibility**: non-macOS builds accept the same config shape but do not attempt path-level Unix socket enforcement, matching the platform limitation.

## Compatibility

No changes to command/exec method names, sandbox policy names, permission profile TOML shape, network proxy environment variables, CLI flags, JSONL event names, TUI behavior, npm package layout, or install commands.

Explicit `deny` Unix socket entries remain parsed configuration. Unlisted sockets stay denied by the restricted macOS sandbox; Linux cannot enforce Unix socket paths by path.

