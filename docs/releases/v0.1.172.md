# Orca v0.1.172

Patch release: expose server-mode shell runtime capabilities for PTY-aware
clients.

## Changes

- **Shell capability query**: server mode now accepts
  `shell/capabilities` and returns a `shell_capabilities` JSONL event.
- **PTY support metadata**: the response reports the current platform, native
  PTY support, PTY resize support, accepted terminal modes, and the terminal
  fallback mode.
- **Command exec constraint metadata**: the response also exposes that
  streaming `command/exec` sessions require a client-supplied `processId`,
  matching the existing protocol validation path.
- **Runtime-owned capability source**: shell platform capability detection now
  lives with `shell_session`, giving future Windows ConPTY or other terminal
  backends one stable protocol surface to extend.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, or existing JSONL event names.

Existing `shell/start`, `shell/read`, `shell/list`, `shell/resize`,
`command/exec`, `command/exec/write`, `command/exec/resize`, and
`command/exec/terminate` behavior is unchanged. Clients that do not call
`shell/capabilities` continue to work as before.

Example request:

```json
{ "id": "shell-caps", "method": "shell/capabilities", "params": {} }
```

Example response shape:

```json
{
  "id": "shell-caps",
  "event": "shell_capabilities",
  "platform": "macos",
  "supportsPty": true,
  "supportsPtyResize": true,
  "supportedTerminalModes": ["pipe", "pty"],
  "fallbackTerminalMode": "pipe",
  "commandExecStreamingRequiresProcessId": true
}
```

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.172 sh
```
