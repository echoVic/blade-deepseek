# Orca v0.1.98

Patch release: move server submit-family dispatch into a focused server
processor module.

## Changes

- **Submit processor boundary**: `server/router.rs` now delegates decoded
  submit-family operations to `server/processors/submit.rs`.
- **Covered operations**: `Submit`, `ThreadStart`, `ThreadResume`, and
  `ThreadFork` now share a focused processor entrypoint.
- **Architecture guardrail**: a new runtime architecture test keeps
  submit-family dispatch details out of the generic router.
- **Behavior preservation**: legacy submit JSONL, thread-bound async turns,
  unknown-thread errors, thread/start, thread/resume, and thread/fork semantics
  are unchanged.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, TUI behavior, install commands, or npm package
layout.

This release completes the current server request-processor extraction sweep.
The next architecture work should focus on deeper TUI loop delegation and
runtime-special dispatch surfaces.
