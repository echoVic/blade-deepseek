# Orca v0.1.64

Patch release: shared commandExecution projection for persisted thread history.

## Changes

- **Shared persisted command items**: persisted bash tool calls now use
  `tool_item_projection` helpers for `commandExecution` history item creation
  and completion instead of hand-written JSON in `thread_store`.
- **History-shape preservation**: the shared persisted helper keeps history-only
  command metadata placeholders such as `cwd`, `processId`, `source`,
  `commandActions`, and `durationMs` while preserving the public
  `commandExecution` item type.
- **Failure semantics preserved**: failed command history items continue to keep
  `aggregatedOutput` empty and place diagnostics in `error`, while successful
  and truncated commands still expose visible output and `truncated` metadata.
- **Boundary coverage**: projection unit tests now lock the persisted
  commandExecution started/completed shape, and the ownership test prevents
  `thread_store` from reintroducing local copies of the shared helpers.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release is an internal projection-boundary refactor for persisted
`commandExecution` thread items.

