# Orca v0.1.42

Patch release: Claude Code-style workflow parity loop and process timeout
cleanup.

## Changes

- **Workflow draft loop**: model-authored workflows can now be persisted as
  drafts with preview metadata, edited, saved, cancelled, cloned from previous
  runs, or launched directly from a `draftId`.
- **Reusable workflow commands**: saved project and user workflows resolve as
  command-like assets with collision-safe aliases and argument schema
  validation before launch.
- **Run controls**: workflow runs now support pause, resume, stop, clone,
  restart failed agents, and restart from a selected phase while preserving
  durable run state.
- **Evidence-bound reporting**: workflow status lines, final summaries,
  verifier output, required evidence checks, failed tool checks, MCP failure
  checks, mutation-policy checks, and child tool events now flow through
  deterministic evidence contracts.
- **TUI and CLI management**: the TUI bridge surfaces draft actions, saved
  workflow args, and richer workflow run metadata; the CLI exposes workflow
  history, source, pause/resume, clone, and restart commands.
- **Process timeout cleanup**: bash tools, external tools, hooks, sandboxed
  commands, and verifier commands now share non-interactive process setup and
  terminate the full child process tree on timeout.

## Compatibility

No changes to install commands, npm package layout, JSONL event names, or the
existing workflow script DSL. Saved workflow argument validation is stricter
when a workflow declares an args schema.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.42 sh
```
