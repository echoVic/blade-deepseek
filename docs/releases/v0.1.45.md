# Orca v0.1.45

Patch release: JSON Schema composition validation for tool arguments.

## Changes

- **`oneOf` validation**: runtime tool-argument validation now rejects calls
  that match zero or multiple `oneOf` branches before the tool executes.
- **`anyOf` validation**: runtime validation now requires at least one `anyOf`
  branch to match and reports branch-level failure details when none do.
- **Object keyword coverage**: `required`, `properties`, and
  `additionalProperties` now apply to object-valued schemas even when a branch
  omits an explicit `"type": "object"`, matching common JSON Schema usage.
- **Provider/runtime alignment**: provider-advertised schemas that use
  composition keywords now have matching pre-execution runtime checks.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm package
layout, install commands, or successful tool-call output formats. Invalid tool
arguments that previously reached tool-specific parsing may now be rejected
earlier by the shared schema gate.
