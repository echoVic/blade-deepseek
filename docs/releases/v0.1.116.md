# Orca v0.1.116

Patch release: continue the P0 runtime turn-loop split by moving the outer
turn-loop orchestration out of `lifecycle.rs` and into a focused
`runtime_turn_loop` module.

## Changes

- **Dedicated turn-loop module**: `RuntimeTurnLoopStep`,
  `RuntimeTurnLoopInput`, and `RuntimeTurnLoopExecutors` now live in
  `runtime_turn_loop.rs` instead of `lifecycle.rs`.
- **Behavior-compatible loop orchestration**: `agent_loop` still delegates
  through `RuntimeTurnLoopStep`, and iteration retry/return folding still uses
  the same `RuntimeTurnIterationStep` behavior.
- **Smaller lifecycle surface**: `lifecycle.rs` no longer owns the outer loop
  step/input/executor grouping, which prepares the next reducer-style runtime
  split inspired by Codex and package 3.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, tool
execution, approvals, workflows, shell sessions, and history persistence remain
unchanged.
