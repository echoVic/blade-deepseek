# Orca v0.1.173

Patch release: add bounded server-mode shell reads for long-running shell
sessions.

## Changes

- **Shell read output caps**: `shell/read` now accepts `outputBytesCap` and
  applies that byte budget to stdout and stderr returned by the read request.
- **Capped stream metadata**: `shell_output_delta` now includes `capReached`,
  matching the existing command/exec streaming metadata shape for clients that
  need bounded reads.
- **Capped shell responses**: `shell_updated` and `shell_completed` include
  `capReached` when a read request supplied `outputBytesCap`, so clients can
  distinguish complete output from budgeted output.
- **UTF-8-safe truncation**: capped shell output is truncated on valid UTF-8
  boundaries instead of splitting a multibyte character.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, or existing JSONL event names.

Existing `shell/read` requests without `outputBytesCap` keep returning the same
stdout/stderr content as before. Existing shell control operations, PTY
selection, `shell/capabilities`, `command/exec`, `command/exec/write`,
`command/exec/resize`, and `command/exec/terminate` behavior is unchanged.

Example request:

```json
{
  "id": "shell-read",
  "method": "shell/read",
  "params": {
    "shellId": "shell-1",
    "timeoutMs": 5000,
    "outputBytesCap": 4096
  }
}
```

Example capped delta:

```json
{
  "id": "shell-read",
  "event": "shell_output_delta",
  "shellId": "shell-1",
  "stream": "stdout",
  "delta": "truncated output",
  "capReached": true,
  "final": false
}
```

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.173 sh
```
