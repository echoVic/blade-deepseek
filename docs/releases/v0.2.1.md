# Orca v0.2.1

Patch release: move server interactive response ownership into focused
processors after another Codex/package 3 architecture pass.

## Changes

- **Permission response ownership**: `permission/respond` now resolves pending
  permission requests inside the server permission processor instead of the
  generic server module.
- **User-input response ownership**: `user_input/respond` now resolves pending
  runtime user-input requests inside the server user-input processor.
- **Architecture guardrails**: ownership tests now fail if response handlers
  drift back into `server.rs`, keeping the server router/processor split closer
  to Codex's request processor shape and package 3's request-id-driven pending
  response flow.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire event names, or JSONL event names.

This release is a behavioral no-op refactor for clients. It keeps permission
and user-input response semantics tied to the same request ids while shrinking
the generic server module.
