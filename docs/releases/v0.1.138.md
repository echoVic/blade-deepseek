# Orca v0.1.138

Patch release: isolate the child-agent loop setup boundary so the
`agent_child` module keeps compatibility re-exports while setup state, provider
bootstrap, conversation seeding, approval policy seeding, and child turn-budget
state live in a focused module.

## Changes

- **Focused child-agent setup module**: `child_agent_loop_setup.rs` now owns
  `ChildAgentLoopSetup`, `ChildAgentTurnBudget`,
  `DEFAULT_CHILD_AGENT_MAX_TURNS`, `prepare_child_agent_loop`, and child turn
  advancement helpers.
- **Compatibility re-export**: existing `orca_runtime::agent_child` imports for
  child-agent setup and turn-budget APIs remain available via re-export.
- **Agent-child shrink**: `agent_child.rs` no longer owns provider bootstrap,
  setup conversation construction, approval policy seeding, or child turn-limit
  state.
- **Ownership coverage**: a new architecture test locks the setup boundary so
  the `agent_child` facade does not silently reacquire setup details.

## Compatibility

No changes to CLI flags, JSONL event names, subagent tool schema, workflow child
agent calls, async subagent workers, TUI subagent execution, model override
behavior, child provider routing, child provider turns, compaction behavior,
tool-call extraction, tool-result folding, cost tracking, approval policy
semantics, MCP routing, external tool routing, shell session protocol,
command/exec wire shape, conversation storage, history storage, npm package
layout, or install commands.

This release is an internal ownership slice. Child-agent behavior for subagent
tool calls, workflow child agents, async subagents, TUI child-agent delegation,
provider bootstrap, model routing, prompt-too-long recovery, and turn-budget
exhaustion is unchanged; the setup boundary is smaller and easier to keep
aligned with the rest of the Codex/package-3-inspired runtime split.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.138 sh
```
