# Orca v0.2.43

Patch release: scope the Linux fail-closed default to strict restricted-read
policies without changing the existing permission, capability-mode, or
managed-proxy contracts.

## Changes

- **Strict-scoped fail closed**: only strict restricted-read policies must
  refuse to run when neither bubblewrap nor Landlock can enforce them. This
  keeps untrusted folders and strict read-only modes fail closed while leaving
  the enforcing backends unchanged.
- **Non-strict compatibility fallback**: workspace-write and global read-only
  modes keep their established Landlock-plus-seccomp or plain-shell fallback
  when a policy needs bwrap-only features (nested read-only or denied paths, or
  Unix-socket exceptions with the network disabled) and no `bwrap` is on
  `PATH`. This matches the reference agents' fail-open-for-built-ins behavior.
- **Linux CI backend**: the release test runner no longer installs bubblewrap,
  so continuous integration exercises the Landlock plus seccomp fallback path
  directly instead of the namespace backend.

## Compatibility

Existing approval rules, capability modes, permission profiles, managed network
proxy behavior, folder trust decisions, server JSONL shapes, CLI arguments, TUI
flows, and persisted history remain compatible. Only the no-backend behavior of
non-strict capability modes changes, and only on Linux; strict restricted-read
policies still fail closed.
