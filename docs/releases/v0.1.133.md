# Orca v0.1.133

Patch release: continue the Codex/package-3-inspired long-argument cleanup by
grouping sandbox bash command construction behind explicit context types.

## Changes

- **Grouped sandbox command contexts**: `orca_tools::sandbox` now exposes
  `WorkspaceWriteSandboxCommandContext` and `ReadOnlySandboxCommandContext`
  instead of long public command-constructor argument lists.
- **Focused Seatbelt profile contexts**: macOS Seatbelt workspace-write and
  read-only profile generation now receives `WorkspaceWriteProfileContext` and
  `ReadOnlyProfileContext`, keeping lower-level profile inputs explicit without
  another helper escape hatch.
- **Runtime call-site cleanup**: shell-session spawning now constructs grouped
  sandbox contexts directly for workspace-write and read-only modes, including
  readable roots, writable roots, denied roots, network flags, tmp exclusions,
  and Unix socket allowlists.
- **Behavior-compatible flow**: shell sessions, command/exec, model-visible bash,
  sandbox profile generation, network and filesystem policy flags, Unix socket
  allowlists, non-interactive process preparation, task registry updates, and
  cancellation behavior keep the same runtime shape.

## Compatibility

No changes to CLI flags, JSONL event names, bash tool input/output shape, shell
session protocol, command/exec wire shape, permission-profile configuration,
network allow/deny semantics, filesystem sandbox grants, Unix socket allowlist
semantics, MCP routing, external tool routing, conversation storage, history
storage, workflows, TUI rendering, npm package layout, or install commands.

This release is an internal architecture slice. Sandbox command construction is
easier to extend and test while preserving the existing runtime behavior.

## Install

```bash

```

Direct binary install:

```bash

```

```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.133 sh
```
