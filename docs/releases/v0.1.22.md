# Orca v0.1.22

Release verification and goal-runtime hardening release.

## Changes

- Add `scripts/release/verify-published.mjs` plus a release-workflow gate that checks the GitHub Release, npm registry version, and `npm exec --package @blade-ai/orca@<version> -- orca --version` after publish.
- Split goal-mode model tools into `get_goal`, `create_goal`, and `update_goal`, with `update_goal` restricted to `complete` and `blocked`.
- Harden goal continuation instructions with evidence-based completion audits, strict blocked audits, progress visibility, and fidelity rules inspired by Codex goal mode.
- Keep release metadata aligned across Cargo, README, roadmap, website metadata, and release notes.

## Verification

```bash
cargo test
npm --prefix site run build
npm --prefix site run check:seo
node scripts/release/test-stage-npm.mjs
node scripts/release/test-verify-published.mjs
git diff --check
```

After publishing, verify the public artifacts:

```bash
node scripts/release/verify-published.mjs --version 0.1.22 --repo echoVic/blade-deepseek --package @blade-ai/orca --bin orca
```

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```
