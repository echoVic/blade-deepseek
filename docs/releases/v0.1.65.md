# Orca v0.1.65

Patch release: persisted file-change history now uses Codex-style projection.

## Changes

- **Persisted fileChange items**: persisted `edit` and `write_file` tool calls
  now project as first-class `fileChange` thread items instead of generic
  `dynamicToolCall` items when reading stored thread history.
- **Realtime/history alignment**: thread-read history now uses the same
  `fileChange` shape already used by realtime server streams, including
  stable `:file-change` item ids, `changes[].path`, `changes[].kind`, and an
  empty diff placeholder.
- **Shared projection boundary**: persisted file-change started/completed item
  creation now lives in `tool_item_projection`, keeping `thread_store` focused
  on storage projection and result merging.
- **Regression coverage**: contract tests cover both persisted `edit` and
  `write_file` replay, and the ownership guard prevents local copies of the
  shared file-change projection helpers.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public realtime event names.
Stored `edit` and `write_file` turns now read back as `fileChange` items instead
of `dynamicToolCall` items, matching the realtime server item model.

## Install



```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.65 sh
```
