# Orca v0.2.6

Patch release: make DeepSeek proposed plans visible and stable in the TUI.

## Changes

- **Dedicated TUI proposed-plan messages**: DeepSeek `<proposed_plan>` streams
  now render as their own scrollback message instead of leaking XML-like tags
  into ordinary assistant text.
- **Shared parser boundary**: server projection and TUI state now use the same
  `orca-core` proposed-plan stream parser, reducing drift between local TUI
  runs and server-facing transcript projection.
- **UTF-8-safe streaming**: split proposed-plan tags, incomplete plan tags, and
  non-ASCII assistant text are covered by parser tests so Chinese streaming
  output does not panic while Orca waits for possible plan tags.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, or JSONL event names.

This release only changes how proposed-plan text is parsed and presented in the
TUI. Existing server proposed-plan item lifecycle events keep their wire shape.
