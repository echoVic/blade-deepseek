# Orca v0.1.34

Patch release: real API e2e release gate.

## Changes

- **Reusable real API gate**: add `scripts/release/real-api-e2e.mjs` so release
  prep can run the provider summary harness, CLI JSONL smoke, and server-mode
  smoke through one repeatable command.
- **Sentinel validation**: the gate parses streamed JSONL output and requires
  the exact assistant sentinels `ORCA_REAL_E2E_OK` and
  `ORCA_SERVER_REAL_OK`, plus successful completion events.
- **Release-script coverage**: add `scripts/release/test-real-api-e2e.mjs`
  with fake `cargo` and `orca` binaries so command wiring and sentinel failure
  behavior stay covered without consuming API quota.
- **Roadmap gate update**: production completion gates now require the real API
  e2e script before tagging a release.

## Compatibility

This release does not change Orca runtime behavior, public CLI flags, JSONL
event names, server-mode wire shapes, npm package layout, or install commands.

## Verification

```bash
cargo fmt -- --check
cargo test --workspace --all-targets
npm --prefix site run build
npm --prefix site run check:seo
node scripts/release/test-stage-npm.mjs
node scripts/release/test-verify-published.mjs
node scripts/release/test-real-api-e2e.mjs
node scripts/release/real-api-e2e.mjs
git diff --check
```

The real API gate requires `DEEPSEEK_API_KEY` or `~/.orca/auth.json` and makes
billable DeepSeek calls.

After publishing, verify the public artifacts:

```bash
node scripts/release/verify-published.mjs --version 0.1.34 --repo echoVic/blade-deepseek --package @blade-ai/orca --bin orca
```

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.34 sh
```
