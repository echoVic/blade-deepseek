# Orca v0.1.177

Patch release: enrich server-mode `command/exec/list` snapshots with shell task
and terminal metadata.

## Changes

- **Shell task identity**: each active `command/exec/list` process snapshot now
  includes the backing `shellId` and `taskId` from the runtime shell-session
  manager.
- **Terminal recovery metadata**: list snapshots now include
  `requestedTerminalMode` and `effectiveTerminalMode`, matching the semantics
  already exposed by `shell/list`.
- **Status parity**: active command exec list responses use the backing shell
  session status when it is available, while still preserving the existing
  `starting` fallback before a process has been activated.
- **Reconnect-ready snapshots**: app-server clients can now restore display
  state and PTY or pipe expectations for active `command/exec` handles without
  separately querying shell sessions.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, or existing JSONL event names.

Existing `command/exec/list` fields remain present. The new `shellId`,
`taskId`, `requestedTerminalMode`, and `effectiveTerminalMode` fields are
additive. Existing `command/exec`, `command/exec/read`,
`command/exec/write`, `command/exec/resize`, `command/exec/terminate`,
`shell/list`, `shell/read`, `shell/capabilities`, and PTY behavior is
unchanged.

Example response:

```json
{
  "event": "command_exec_listed",
  "id": "cmd-list",
  "processes": [
    {
      "processId": "process-1",
      "shellId": "shell-1",

      "command": ["sh", "-lc", "sleep 30"],
      "cwd": "/workspace",
      "status": "running",
      "requestedTerminalMode": "pipe",
      "effectiveTerminalMode": "pipe",
      "streamOutput": true,
      "outputBytesCap": 4096,
      "stdoutBytes": 12,
      "stderrBytes": 0
    }
  ]
}
```
