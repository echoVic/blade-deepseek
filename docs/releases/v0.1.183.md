# Orca v0.1.183

Patch release: expose runtime capability snapshots as the directive-state contract.

## Changes

- **Named runtime capability contract**: `RuntimeCapabilityPatch` and
  `RuntimeCapabilitySnapshot` now own the model override, allowed-tool
  replacement, runtime system-message injection, and transition-reason state
  behind runtime directives.
- **Directive-state wrapper**: `RuntimeDirectiveState` applies capability
  patches and exposes the shared capability snapshot instead of owning a
  parallel set of runtime fields.
- **Reducer integration**: `RuntimeTurnReducer` can apply capability patches to
  a snapshot directly, giving future skill, hook, MCP, and tool-policy paths a
  typed state mutation surface.
- **Architecture guardrail**: focused runtime tests prevent directive state from
  drifting away from the named capability snapshot contract.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, or JSONL event names.

Runtime turn setup, provider response handling, tool dispatch, approval
overlays, workflow tools, subagent batching, goal progress, and history
recording remain behavior preserving.
