# Orca v0.1.182

Patch release: assemble turn-loop state through a runtime turn kernel instance.

## Changes

- **Instance-owned loop-state handoff**: `RuntimeTurnKernel` now exposes
  turn-loop state assembly as an instance method instead of a static helper.
- **Kernel-created from scoped stores**: `RuntimeTurnState` creates the kernel
  from the current thread and turn extension stores before asking it to build
  `RuntimeTurnLoopState`.
- **Shared turn extension store**: turn-scoped extension data is now shared
  through `Arc<ExtensionData>`, allowing the kernel to borrow the stores it uses
  while the loop state receives the same scoped stores for execution.
- **Architecture guardrail**: a focused runtime test prevents the lifecycle
  path from drifting back to static loop-state assembly or direct
  `RuntimeTurnLoopState` construction.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, or JSONL event names.

Runtime turn setup, provider response handling, tool dispatch, approval
overlays, workflow tools, subagent batching, goal progress, and history
recording remain behavior preserving.
