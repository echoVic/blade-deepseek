# Orca v0.2.5

Patch release: make server command/exec network-policy denials explicit.

## Changes

- **Server command/exec denial clarity**: command/exec now keeps denylisted
  network block reports and maps them through the shared runtime permission
  evaluation path, producing a clear policy-denial error.
- **Prompt behavior preserved**: requestable allowlist/private-policy blocks
  still produce the existing permission request and retry flow.
- **Cross-surface permission parity**: TUI bash and server command/exec now
  share the same `Request` or `Deny` network-block classification, reducing
  drift between local TUI runs and remote/server clients.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, or JSONL event names.

This release only changes final-denial reporting for configured command/exec
network denylist blocks. Requestable command/exec network prompts keep the same
permission request shape.
