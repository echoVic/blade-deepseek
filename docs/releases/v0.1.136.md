# Orca v0.1.136

Patch release: isolate the runtime permission boundary so lifecycle
coordination keeps its compatibility exports while permission requests,
responses, handlers, and the turn-scoped permission overlay live in a focused
module.

## Changes

- **Focused runtime permission module**: `runtime_permission.rs` now owns
  `RuntimePermissionRequest`, `RuntimePermissionResponse`,
  `RuntimePermissionRequestHandler`, `AllowRequestedPermissions`, and
  `TurnPermissionOverlay`.
- **Compatibility re-export**: existing `orca_runtime::lifecycle` imports for
  runtime permission request/response/handler and turn permission overlay types
  remain available via re-export.
- **Lifecycle shrink**: `lifecycle.rs` no longer owns permission request and
  overlay internals, keeping it focused on runtime coordination and delegation.
- **Ownership coverage**: a new architecture test locks the module boundary so
  `lifecycle.rs` does not silently reacquire permission request and overlay
  details.

## Compatibility

No changes to CLI flags, JSONL event names, `request_permissions` tool schema,
permission response decisions, turn/session grant semantics, strict auto-review
merging, bash tool input/output shape, shell session protocol, command/exec wire
shape, permission-profile configuration, network allow/deny behavior,
filesystem sandbox grants, Unix socket allowlist semantics, MCP routing,
external tool routing, conversation storage, history storage, workflows, TUI
rendering, npm package layout, or install commands.

This release is an internal ownership slice. Runtime behavior for
`request_permissions`, command/exec permission retries, model-visible bash
permission retries, and turn permission overlays is unchanged; the boundary is
smaller and easier to keep aligned with the rest of the Codex/package-3-inspired
runtime split.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.136 sh
```
