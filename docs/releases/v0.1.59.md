# Orca v0.1.59

Patch release: shared completed-item projection plus release-runner MCP fixture hardening.

## Changes

- **Shared completed items**: realtime app-server streams and persisted thread
  projections now use the same `tool_item_projection` builders for completed
  `mcpToolCall` and `dynamicToolCall` items.
- **Schema drift guardrail**: the ownership test now prevents
  `server_runtime` and `thread_store` from reintroducing separate MCP/dynamic
  completed-item builders.
- **Failure-shape coverage**: projection tests lock successful and failed MCP
  and dynamic completed-item shapes, including result/content item slots,
  success booleans, and normalized `exitCode` errors.
- **Command compatibility guard**: persisted history now has explicit coverage
  that failed `commandExecution` projections preserve error metadata without
  accidentally writing failed output into `aggregatedOutput`.
- **CI fixture hardening**: stdio MCP shell fixtures now launch through
  `/bin/sh <script>` instead of directly execing freshly written temporary
  scripts, avoiding Linux `Text file busy (os error 26)` release flakes.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release is an internal projection-boundary refactor for first-class
`mcpToolCall` and `dynamicToolCall` completed items, plus a release-test
hardening patch.

## Install

```bash
npm install -g @blade-ai/orca

```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.59 sh
```
