# Orca v0.2.12

Patch release: overhaul TUI scroll performance so long sessions stay responsive.

## Changes

- **Frame scheduler**: a new frame scheduler coalesces wheel events and caps how
  many events are processed per batch, so rapid scrolling no longer floods the
  render loop.
- **Message render caching**: rendering now flows through unified message
  management with a message-version-based cache, so scrolling no longer redraws
  the full transcript every frame.
- **Virtual viewport**: the transcript renders only the messages in the visible
  region instead of the entire history.
- **Wider scroll metrics**: scroll offsets are now `usize`, so sessions longer
  than 65,535 lines scroll correctly.
- **Cleaner status line**: the bottom status line no longer shows the
  `scroll: N/total` indicator; it now starts directly with the model metadata.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, approval options, or shortcut behavior.

This release only changes internal TUI rendering and scroll handling. Existing
key combinations and on-screen content keep the same meaning; the visible
difference is smoother scrolling and a slightly leaner status line.

```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.12 sh
```
