# Orca v0.1.122

Patch release: continue the P0 runtime reducer split by moving conversation
bootstrap out of `lifecycle.rs` and into a focused
`runtime_conversation_bootstrap` module.

## Changes

- **Dedicated conversation bootstrap module**:
  `RuntimeConversationBootstrapStep` now lives in
  `runtime_conversation_bootstrap.rs` instead of `lifecycle.rs`.
- **Prepared conversation boundary moved with the step**:
  `RuntimePreparedConversation` and its borrowed-or-owned conversation storage
  now live next to the bootstrap step that creates them.
- **Preserved agent-loop behavior**: `run_agent_loop` still delegates through
  `RuntimeConversationBootstrapStep`, still composes the session-owned
  conversation bootstrap path, and still records initial history before the
  first model turn.
- **Smaller lifecycle surface**: `lifecycle.rs` keeps actor/lifecycle
  primitives and turn setup while shedding another reducer-sized orchestration
  slice.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, initial
conversation creation, resumed conversation handling, initial history recording,
model routing, compaction, steer injection, tool execution, approvals,
workflows, shell sessions, and history persistence remain unchanged.
