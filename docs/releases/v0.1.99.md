# Orca v0.1.99

Patch release: move runtime-special tool dispatch into a focused runtime
module.

## Changes

- **Runtime-special boundary**: `RuntimeToolActorContext` now keeps
  runtime-special dispatch classification and small executors in
  `runtime_special.rs` instead of `lifecycle.rs`.
- **Covered tools**: `request_permissions`, workflow IPC, `subagent_status`,
  `task_list`, `task_stop`, and workflow draft preview execution preserve their
  existing behavior behind the new module boundary.
- **Architecture guardrail**: a new runtime architecture test keeps
  runtime-special dispatch details out of `lifecycle.rs`.
- **Behavior preservation**: permission overlay merging, workflow child-agent
  IPC errors, subagent status projection, task list/stop output shape, and
  workflow draft serialization are unchanged.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, TUI behavior, install commands, or npm package
layout.

This release finishes the current runtime-special extraction slice. The next
architecture work should continue deeper TUI loop delegation and package-3-style
runtime turn decomposition.
