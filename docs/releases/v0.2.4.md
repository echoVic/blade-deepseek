# Orca v0.2.4

Patch release: make TUI bash network-policy denials explicit.

## Changes

- **Explicit network denials**: runtime permission policy now returns a
  structured `Request` or `Deny` evaluation for bash/command network blocks,
  so denylisted hosts carry a stable final-denial reason instead of being
  represented only as a missing prompt.
- **TUI bash reliability**: TUI bash execution now consumes that shared
  evaluation path. Requestable network blocks still open the normal approval
  flow, while configured denylist blocks end as a clear denied tool result.
- **Ownership guardrails**: runtime and TUI ownership tests now guard the new
  permission-evaluation boundary so future approval work stays out of
  UI-local ad hoc policy code.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire event names, or JSONL event names.

This release only improves how TUI bash network denials are classified and
reported. Existing requestable network permission prompts keep the same approval
flow.
