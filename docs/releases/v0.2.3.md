# Orca v0.2.3

Patch release: route real stdio MCP elicitation requests through the TUI
pending-interaction path.

## Changes

- **TUI-visible MCP elicitation**: stdio MCP servers can now send
  `elicitation/create` during a tool call and have Orca surface the request as a
  TUI waiting-input prompt instead of silently dropping the message.
- **Runtime id discipline**: MCP elicitation requests reuse the runtime pending
  interaction store, preserving server name, request id, URL/form mode, URL, and
  schema metadata until the user answers or cancels.
- **Transport response routing**: accepted TUI responses are written back to the
  MCP server as JSON-RPC accept results; cancelled or declined prompts return a
  decline result and the original tool call can continue or fail according to the
  server's response.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, hook
schema, workflow scripts, npm package layout, install commands, server-mode wire
event names, or JSONL event names.

The new elicitation handling currently covers stdio MCP transports. SSE MCP
tool calls keep their previous request/response behavior until Orca has an
equivalent streaming request channel for SSE.
