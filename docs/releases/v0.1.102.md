# Orca v0.1.102

Patch release: move TUI child-agent execution behind runtime-owned child-agent
facades while preserving the v0.1.101 reasoning-effort flow.

## Changes

- **Runtime-owned TUI child-agent loop**: TUI child subagent runs now delegate
  request construction, model/cost setup, provider selection, turn advancement,
  compaction, provider calls, provider-error handling, response folding, tool
  request extraction, and tool-result folding to `orca-runtime`.
- **Narrow TUI adapter surface**: the TUI now supplies only the interactive
  tool-executor adapter. Runtime passes that adapter a narrow tool context for
  approval policy and MCP registry access instead of exposing child loop setup
  internals.
- **Request construction facade**: TUI child runs now pass prompt, subagent
  type, model override, and depth into a runtime prompt-level facade instead of
  constructing `ChildAgentRequest` directly.
- **Reasoning-effort preservation**: child provider configs continue to receive
  the configured DeepSeek reasoning effort after the setup moved into runtime.
- **Architecture guardrails**: focused TUI boundary tests now prevent
  `agent_subagent_execution` from re-owning child provider turns, provider
  response folding, tool request extraction, tool result folding, turn state, or
  child request construction.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, or
model-visible subagent result text.

This release is an internal architecture slice toward a runtime-owned
`Session -> Task -> Turn` child-agent boundary. TUI behavior is intended to stay
unchanged while the implementation surface gets smaller and easier to verify.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.102 sh
```
