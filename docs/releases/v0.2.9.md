# Orca v0.2.9

Patch release: make TUI main-agent compaction and prompt-too-long recovery
runtime-owned.

## Changes

- **Runtime-owned TUI compaction**: the main TUI loop now enters
  `orca-runtime` for pre-turn automatic compaction instead of directly reading
  provider context pressure or calling summary compaction helpers itself.
- **Runtime-owned prompt-too-long retry**: provider context-length errors in the
  TUI main-agent path now go through the runtime compaction retry policy, using
  the same one-retry guard and compaction event path as other runtime-owned
  provider turns.
- **Stable TUI visibility**: context usage updates, compacted-context notices,
  failed compaction errors, task status, and session completion still use the
  existing TUI event shapes, but the renderer loop no longer owns the policy
  that decides when to compact or retry.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode method names, JSONL event names, approval prompts, task
panel actions, or current TUI context-status rendering.

This release only changes ownership of TUI main-agent compaction policy and
retry state so long TUI sessions behave more like the server and child-agent
runtime paths.
