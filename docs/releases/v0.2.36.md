# Orca v0.2.36

Feature release: reliably interrupt foreground subagents through one
runtime-owned lifecycle while keeping durable background tasks independently
visible and stoppable in the TUI.

## Why This Release

Synchronous single and batch subagents previously used two execution owners: an
inline child loop for one call and a scoped thread runtime for a batch. Child
cancellation, joins, panic conversion, schema validation, worktree cleanup, and
terminal formatting therefore had no single boundary. A child panic could
escape RuntimeHost, and an interrupted foreground turn could not prove every
started child had been joined before the TUI admitted the next prompt.

Async delegation had a separate consistency defect. It emitted a foreground
`subagent.started` event even though ownership moved to a durable task, leaving
an unmatched TUI row. Parent PID adoption could also overwrite newer progress
or terminal state from a fast worker.

These were lifecycle and data-ownership defects, not isolated cancellation or
presentation bugs. Patching each symptom would have preserved duplicate
runtimes and competing task-state writers.

## Changes

- **One synchronous subagent owner**: `RuntimeToolCallRuntime` owns admission,
  lifecycle start, child cancellation, worker spawn, join, panic classification,
  schema validation, usage, worktree completion, result formatting, and the
  exactly-once terminal for both single and batch delegation.
- **Reliable interrupt and recovery**: interrupt reaches every admitted child,
  prevents later admission, waits for all worker and worktree cleanup, preserves
  provider request order, and returns RuntimeHost to a clean next prompt.
- **Panic isolation**: a child panic after admission becomes one indeterminate
  tool result and one failed subagent lifecycle terminal instead of escaping the
  host. Worktree cleanup still runs after the panic.
- **Immediate durable task projection**: async launch emits the existing typed
  `task.status.updated` event so the TUI task panel can show and stop the task
  immediately, without an unmatched foreground `subagent.started` row.
- **Ordered async ownership handoff**: `TaskRegistry` atomically persists the
  real worker PID, Running state, and start time before the worker may write
  progress or terminal state. Late adoption cannot overwrite a terminal task,
  and the parent reaper reloads persisted worker completion.
- **Independent cancellation domains**: foreground interrupt does not stop a
  registered async worker. Durable cancellation remains explicit through
  `task_stop` and the TUI task panel.
- **Old paths deleted**: the inline single-child loop, scoped batch runtime,
  duplicate schema/worktree/result formatting, stale parent adoption path,
  generic foreground child-executor plumbing, and related source-shape tests
  are gone.

## Compatibility

CLI arguments, TUI keys and foreground flow, server methods and JSONL wrappers,
tool names and schemas, synchronous `subagent.started` / `subagent.completed`
payloads, provider call ids and request ordering, persisted task fields,
`subagent_status`, `task_stop`, worktree messages, and DeepSeek request behavior
remain compatible. Async calls now expose their existing durable task event
instead of an invalid foreground lifecycle. Cross-process leases, fencing,
stale-owner takeover, and task-wide live publication remain the explicit P1.4
task-supervisor boundary.

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.36
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.36 sh
```
