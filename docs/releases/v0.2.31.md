# Orca v0.2.31

Feature release: keep recorded conversation rows stable across restart and
resume, and separate logical turn control from runtime task identity.

## Why

Saved thread turns and items were previously named from their current vector
positions. Compaction, compatibility repair, filtering, or a later projector
change could therefore rename a row that a TUI or server client had already
observed. Server active-turn routing also reused that projected label as a
process-global key, so two threads could both predict `turn-1` and address the
wrong operation.

This release moves identity to the durable ownership boundary. A logical turn
gets one opaque id before runtime admission, and each new conversation record
stores its own item id plus the owning turn id.

## Changes

- **Stable saved rows**: reload, resume, pagination, compaction, compatibility
  repair, redaction, rename, archive, and zstd restoration preserve the ids of
  surviving turns and items.
- **Correct control routing**: logical turn ids are distinct from operation,
  runtime task, and TaskRegistry ids. Concurrent threads cannot collide merely
  because both are on their first turn.
- **One persistence source**: `SessionWriter` owns identity assignment and
  updates its shared ledger only after a successful append. Model replay remains
  a derived view of those same records.
- **Domain-native tool identity**: tool items retain the DeepSeek tool-call id,
  file-change projections retain the `:file-change` suffix, workflows retain
  their run id, and tool results update the original item.
- **Bounded compatibility**: old histories remain readable through one named
  legacy fallback. New records cannot mint `turn-N` or `item-N`, and malformed
  partial identity fails closed.
- **Real restart proof**: the release harness now records a live DeepSeek turn,
  cold-reads its ids, exits, resumes the same thread in a second process, and
  verifies both context continuity and exact preservation of the old id prefix.

## Compatibility

CLI arguments, TUI keys and flows, server method names, existing response field
names, JSONL event names, model-visible conversation content, tool/workflow item
shapes, and existing history files remain compatible.

`turn_started` adds `turnId`, and new `conversation.message` records add `id`
and `turn_id`. These values are opaque protocol identifiers and must not be
parsed as counters. Older records without them continue through the explicit
legacy projector; malformed non-empty identity does not silently downgrade to
legacy.

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.31
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.31 sh
```
