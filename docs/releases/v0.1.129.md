# Orca v0.1.129

Patch release: continue the Codex-style server boundary split by moving
server shell-session state out of `server.rs` and into a focused
`server/shell_manager.rs` module.

## Changes

- **Dedicated shell manager module**: `ServerShellManager` now owns optional
  `RuntimeShellSessionManager` storage, lazy task-registry-backed shell manager
  creation, and the server shell pass-through operations for spawn, stdin,
  close, update, resize, list, reap, read, wait, and kill.
- **Smaller server orchestrator**: `server.rs` keeps JSON protocol decoding,
  server event construction, shell error shaping, and command/exec orchestration
  while delegating shell session storage and lifecycle calls to the manager.
- **Behavior-compatible shell flow**: shell start/write/update/close/resize/list/read/kill
  and command/exec process drain/write/resize/terminate compatibility keep the
  same server protocol shape.
- **Documented P0 slice**: added a focused implementation plan for the shell
  manager boundary so the next server lifecycle split can continue from a
  smaller `server.rs`.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names, shell
session ids, command/exec process ids, permission profile TOML shape, install
commands, npm package layout, provider requests, model selection behavior,
thread ids, turn ids, shell sandbox policy, PTY behavior, command/exec timeout
semantics, or command/exec streaming event shapes.

This release is an internal architecture slice. Server-mode turn execution,
command/exec behavior, shell sessions, workflows, subagents, history projection,
and server/TUI/headless runtime execution remain unchanged.

