# Orca v0.1.115

Patch release: continue the P2 tool-execution boundary cleanup by grouping
model-visible `bash` shell-session execution inputs behind one runtime-owned
context.

## Changes

- **Grouped bash invocation**: `runtime_bash` now exposes
  `RuntimeBashInvocationContext` and `execute_bash_with_shell_session` consumes
  that single context instead of a long flat argument list.
- **Narrower normal-tool handoff**: `RuntimeNormalToolExecutor` still owns the
  bash branch, but now constructs the grouped bash context before delegating to
  the shell-session runner.
- **Behavior-compatible shell path**: permission overlays, cancellation, output
  truncation, task registry handoff, active permission-profile network policy,
  and filesystem/network permission retry behavior remain unchanged.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or approval response wire shapes.

This release is an internal architecture slice that keeps shrinking the bash
execution call surface before deeper shell/session and async-subagent work.
