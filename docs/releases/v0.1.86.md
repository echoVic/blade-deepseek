# Orca v0.1.86

Patch release: make session-scoped network denials from `request_permissions`
tighten later thread-bound `command/exec` network policy.

## Changes

- **Session network overlay precedence**: session-scoped network `deny` entries
  now override permission-profile `allow` entries for the same domain when a
  later thread-bound `command/exec` builds its managed proxy policy.
- **Allow remains conservative**: session-scoped network `allow` entries still
  only fill missing policy entries, so they do not bypass an existing
  permission-profile `deny`.
- **Proxy diagnostics**: denied session overlays surface the Codex-style
  `blocked-by-denylist` proxy reason instead of falling through to a generic
  policy block.
- **Harness coverage**: server contract coverage proves a profile allow can be
  tightened by a later session deny before command execution.

## Compatibility

No changes to command/exec method names, sandbox policy names, permission
profile TOML shape, network proxy environment variables, CLI flags, JSONL event
names, TUI behavior, npm package layout, or install commands.

This release keeps the interactive network-permission model conservative:
session approvals can expand missing allowlist entries, while session denials
can always reduce later thread network access.
