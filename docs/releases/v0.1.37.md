# Orca v0.1.37

Patch release: shell execution timeout hardening.

## Changes

- **Shell timeout config**: `[tools].shell_timeout_secs` now controls the timeout for synchronous `bash` execution and external TOML tools, with a 120s default and normalization to the 1..3600s range.
- **Process wait helper**: shell and external tool execution now share a timeout-aware child process wait helper, so timed-out commands stop cleanly instead of hanging indefinitely.
- **Runtime propagation**: the runtime controller passes the configured timeout through to tool execution, keeping shell behavior consistent across the tool stack.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm package layout, or install commands.
