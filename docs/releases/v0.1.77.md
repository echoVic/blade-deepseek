# Orca v0.1.77

Patch release: seed the runtime-owned thread boundary that will let server, TUI, and headless execution converge on one long-lived runtime surface.

## Changes

- **Runtime thread boundary**: added `orca_runtime::thread::RuntimeThread`, grouping `InteractiveSession` and `RuntimeSessionLifecycle` behind one runtime-owned facade.
- **Turn delegation hook**: `RuntimeThread::run_turn_to_writer` delegates to the existing `ThreadTurnExecutor` path, creating the new boundary without changing current execution behavior.
- **Architecture guard**: new runtime tests prove the thread starts with a runtime-owned session/lifecycle pair and exposes session mutation through the thread boundary.
- **Roadmap alignment**: production roadmap and implementation plan now document the RuntimeThread seed as the next convergence point before deeper protocol/TUI/server migration.

## Compatibility

No changes to CLI flags, server-mode method names, JSON request/response shapes, realtime event names, npm package layout, direct binary install commands, thread storage formats, or TUI behavior.

This release is an architecture slice only. It makes the next RuntimeThread adoption work easier to review while keeping the existing `ThreadTurnExecutor` behavior intact.
