# Orca v0.1.137

Patch release: isolate the runtime approval boundary so lifecycle coordination
keeps its compatibility exports while approval decisions, handlers, and the
config-backed interactive approval adapter live in a focused module.

## Changes

- **Focused runtime approval module**: `runtime_approval.rs` now owns
  `RuntimeApprovalDecision`, `RuntimeApprovalHandler`, and
  `RuntimeConfigApprovalHandler`.
- **Compatibility re-export**: existing `orca_runtime::lifecycle` imports for
  runtime approval decisions and handlers remain available via re-export.
- **Lifecycle shrink**: `lifecycle.rs` no longer owns approval handler
  internals, keeping it focused on runtime coordination and delegation.
- **Ownership coverage**: a new architecture test locks the module boundary so
  `lifecycle.rs` does not silently reacquire approval decision and handler
  details.

## Compatibility

No changes to CLI flags, JSONL event names, approval request schemas, approval
policy resolution, interactive approval behavior, denial result shape,
`request_permissions` tool schema, permission response decisions, turn/session
grant semantics, bash tool input/output shape, shell session protocol,
command/exec wire shape, permission-profile configuration, MCP routing,
external tool routing, conversation storage, history storage, workflows, TUI
rendering, npm package layout, or install commands.

This release is an internal ownership slice. Runtime behavior for approval
allow/ask/deny decisions, config-backed interactive approval resolution,
headless tool execution, and TUI approval handling is unchanged; the boundary is
smaller and easier to keep aligned with the rest of the Codex/package-3-inspired
runtime split.

## Install

```bash

```

```bash

```

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.137 sh
```
