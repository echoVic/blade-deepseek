# Orca v0.1.123

Patch release: continue the P0 runtime reducer split by moving turn setup out
of `lifecycle.rs` and into a focused `runtime_turn_setup` module.

## Changes

- **Dedicated turn setup module**: `RuntimeTurnSetupStep` now lives in
  `runtime_turn_setup.rs` instead of `lifecycle.rs`.
- **Setup output moved with the step**: `RuntimeTurnSetup` now sits next to the
  preparation logic that creates the runtime context config, approval policy,
  and provider config for each model turn.
- **Preserved agent-loop behavior**: `run_agent_loop` still delegates through
  `RuntimeTurnSetupStep`, still builds `ContextConfig` from the configured model
  runtime, still constructs the tool approval policy through the tool-execution
  boundary, and still composes provider config through the tool-invocation
  boundary.
- **Smaller lifecycle surface**: `lifecycle.rs` keeps actor/lifecycle
  primitives and test coverage while shedding another reducer-sized setup
  slice.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, model selection behavior, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, context
budgeting, provider config construction, tool approval policy construction,
model routing, compaction, steer injection, tool execution, approvals,
workflows, shell sessions, and history persistence remain unchanged.
