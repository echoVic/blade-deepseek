# Orca v0.2.49

Orca v0.2.49 rebuilds Goal Mode around one runtime-owned lifecycle so a long-running
goal can stop, recover, resume, and finish without duplicate continuations or
ambiguous terminal state.

## What Changed

- A per-thread `GoalActor` now owns goal state, run and outer-turn ledgers,
  continuation admission, cancellation, verification, usage, and persistence.
  The TUI submits one composite Goal run instead of driving its own continuation
  loop.
- Model calls to `update_goal` now submit typed terminal intents. Completion and
  blocking claims are applied only after turn-end preflight and bounded DeepSeek
  verification; model-fixable gaps pause as no progress instead of being reported
  as external blockers.
- SQLite is the authoritative Goal store. State transitions, usage, intents, and
  in-flight markers commit transactionally; stale in-flight work recovers to a
  paused state after restart. Existing `goals_1.json` data migrates once and is
  backed up only after a successful transaction.
- Pause, cancel, resume, and shutdown route through the same runtime owner and
  wait for active generation cleanup. A resumed Goal receives a fresh run and
  outer-turn identity, while cancellation cannot leave an orphan active run.
- Goal context is rendered as a bounded internal system fragment without
  rewriting transcript messages or tool results. Typed Goal events now drive TUI
  and ACP status projections.
- Ordinary hosted TUI turns retain normal turn semantics when no Goal is active,
  preserving cancellation, foreground/background task metadata, approvals, and
  workflow labels.

## Compatibility

The `/goal`, `/goal <objective>`, `/goal edit`, `/goal pause`, `/goal resume`, and
`/goal clear` commands are unchanged. Existing Goal JSON data migrates
automatically. `orca exec` remains outside the Goal control contract, and the ACP
and internal JSONL wire methods remain compatible.

## Verification

- 1,600 serial library tests across `orca-core`, `orca-tools`, `orca-runtime`, and
  `orca-tui`
- complete serial workspace tests across all targets
- workspace Clippy, Rust formatting, release helper, and website build/SEO checks
- five real DeepSeek Goal scenarios: completion, rejected completion, genuine
  blocked, cancellation, and resume
- event journal and SQLite audit with zero stale continuations and zero in-flight
  runs after every real scenario

## Install

```bash
npm install -g @blade-ai/orca@0.2.49
```

Or install a native release asset:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.49 sh
```
