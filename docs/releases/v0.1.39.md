# Orca v0.1.39

Patch release: workflow orchestration hardening and subagent capabilities.

## Changes

- **Workflow child task list tools**: agents inside a workflow can now query
  sibling task status via dedicated list/get tools exposed through IPC.
- **Typed output schemas for subagents**: subagent spawners may declare a JSON
  Schema for the expected output; the runtime validates the result before
  returning it to the parent.
- **Team tool allowlists**: workflow config gains a per-team `allowed_tools`
  list, enforced by the controller before dispatching any tool call.
- **Durable workflow IPC state**: IPC channels now persist across host restarts,
  with schema-validated message envelopes and replay on reconnect.
- **Agent lifecycle observability**: workflow runner emits structured lifecycle
  events (spawn, complete, fail, cancel) surfaced in the TUI task panel.
- **Roadmap section ordering fix**: docs-only reorder for production roadmap.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm
package layout, or install commands.

## Install

```bash

```

Direct binary install:

```bash

```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.39 sh
```
