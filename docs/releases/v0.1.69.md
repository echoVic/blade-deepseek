# Orca v0.1.69

Patch release: tool-turn execution now lives behind a dedicated runtime module.

## Changes

- **Tool-turn boundary**: Cursoring, readonly batching, normal tool-turn execution, tool result recording, and tool-turn terminal folding moved from `tool_invocation` into `tool_turn`.
- **Narrower invocation module**: `tool_invocation` now stays focused on provider tool schemas, provider tool-call extraction, request preparation, hook-modified validation, and approval request shaping.
- **Ownership tests**: Runtime ownership tests now prevent tool-turn execution details from drifting back into `agent_loop` or `tool_invocation`.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, public realtime event names, or TUI
display behavior.

