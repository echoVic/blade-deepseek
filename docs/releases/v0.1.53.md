# Orca v0.1.53

Patch release: realtime tool item exit-code projection.

## Changes

- **Realtime item error parity**: app-server `mcpToolCall` and
  `dynamicToolCall` item completions now include `error.exitCode` when the
  runtime tool completion payload carries `exit_code`.
- **MCP failure detail alignment**: failed realtime MCP item completions now
  preserve both the user-visible error message and the process-style exit code,
  matching persisted thread item projections.
- **Dynamic tool failure detail alignment**: failed realtime external/dynamic
  tool item completions now expose the same `error.exitCode` shape that
  `thread/items/list` already returns after persistence.
- **Regression coverage**: server writer tests now cover failed realtime MCP
  and dynamic tool item errors with exit codes.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode
methods, npm package layout, install commands, MCP routing, or successful tool
item payloads. Clients that already read `error.message` can keep doing so;
clients that need failure classification can now also read `error.exitCode` on
realtime MCP/dynamic tool item completions when available.
