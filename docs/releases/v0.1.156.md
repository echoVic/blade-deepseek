# Orca v0.1.156

Patch release: route runtime directive application through the runtime-owned
turn reducer.

## Changes

- **Directive reducer branch**: `RuntimeTurnReducer` now applies
  `RuntimeDirective` values to `RuntimeDirectiveState`, alongside the existing
  completed-tool goal-progress reduction path.
- **Runtime turn-state delegation**: `RuntimeTurnState::apply_directive` now
  constructs a reducer over the live thread and turn extension stores instead
  of writing directive state directly.
- **State API tightening**: `RuntimeDirectiveState::apply` is now a crate-local
  reducer primitive rather than a public mutation entry point.
- **Regression coverage**: tests prove model switches, allowed-tool
  replacement, injected system messages, transition reasons, tool-policy
  replacement, and model-route override behavior still work through the new
  reducer boundary.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, TUI display
layout, server wire shapes, app-server events, JSONL event names, thread
history, goal storage format, tool schemas, npm package layout, or install
commands.

This is an internal architecture slice that continues the Codex/package-3
direction: runtime owns turn-state reduction, while callers keep behavior-level
interfaces. It prepares the remaining reducer slices for permission and
turn-control state without changing user-facing behavior.
