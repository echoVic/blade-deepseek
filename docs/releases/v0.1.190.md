# Orca v0.1.190

Patch release: group provider-turn execution inputs behind a named runtime
handoff.

## Changes

- **Provider-turn input object**: `RuntimeProviderTurnStep::run` now receives
  `RuntimeProviderTurnInput` instead of a long flat provider-call parameter
  list.
- **Provider-turn I/O bundle**: conversation, history writer, event factory,
  sink, and cost tracker refs now flow through `RuntimeProviderTurnIo`.
- **Cycle assembly point**: `RuntimeTurnProviderCycleStep` assembles the
  provider-turn input at the boundary where provider-cycle state enters the
  provider call.
- **Architecture coverage**: a source-level regression test locks the boundary
  so future provider-call work keeps the grouped input shape instead of
  reintroducing parallel I/O fields.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, server-mode wire events, JSONL event names, or tool behavior.

This release is an internal runtime architecture slice. Pre/post model hooks,
runtime system-message injection, provider streaming, steering, usage tracking,
history usage appends, cancellation checks, and provider delta emission keep the
same behavior while the provider-turn handoff becomes smaller and better aligned
with Codex-style turn/step context boundaries and Claude Code-style
query/tool context grouping.
