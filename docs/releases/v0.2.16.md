# Orca v0.2.16

Patch release: make automatic and manual context compaction visible while the
TUI is waiting for DeepSeek summary work.

## Changes

- **Compaction started event**: runtime compaction emits the typed
  `context.compaction.started` event with the trigger reason and pre-compaction
  message count.
- **Early lifecycle ordering**: the started event is emitted before budget
  warning hooks, pre-compact hooks, or remote summary work can block.
- **TUI projection**: runtime started events enter the existing
  `Compacting context...` activity state immediately.
- **Manual parity**: `/compact` emits the same started state before its
  synchronous summary call and returns to idle after the compacted notice.
- **Real interruption**: `Ctrl+C`, `Esc`, and `Ctrl+G` continue to interrupt
  while the TUI is in `Compacting context...`; the same cancellation token now
  reaches compaction hooks and the streaming DeepSeek summary request.
- **Truthful completion**: `context.compacted` is emitted only after summary
  state persistence and the post-compact hook finish, so the TUI does not return
  to idle while compaction work is still blocked.
- **Typed payloads**: started and completed compaction payloads share typed core
  structs before the existing JSONL wire serialization boundary.
- **Wire compatibility**: older `context.compacted` envelopes that contain only
  before/after message counts still project into the TUI with empty optional
  detail fields.
- **Completion detail preserved**: completed compactions still report reason,
  strategy, before/after message counts, and collapsed-message totals.

## TUI impact

Long DeepSeek summary requests no longer look like a frozen turn. Users see
`Compacting context...` for automatic soft-limit, hard-limit, prompt-too-long
recovery, and manual compaction from the beginning of the potentially slow
operation.

Compaction remains controllable while that status is visible. Interrupting a
remote DeepSeek summary cancels the streaming read and lets the turn unwind
through the existing cancelled-session path.

## Compatibility

No changes to CLI flags, slash-command syntax, model/provider configuration,
MCP configuration, hooks, workflows, npm package layout, approvals, shortcuts,
or the existing `context.compacted` payload.

The JSONL runtime event stream adds `context.compaction.started`. Existing
consumers that ignore unknown event types remain compatible, and the TUI keeps
accepting the earlier minimal `context.compacted` payload.

```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.16 sh
```
