# Orca v0.2.16

Patch release: make automatic and manual context compaction visible while the
TUI is waiting for DeepSeek summary work, and make DeepSeek streaming
transport failures recoverable without replaying partial output.

## Changes

- **Compaction started event**: runtime compaction emits the typed
  `context.compaction.started` event with the trigger reason and pre-compaction
  message count.
- **Early lifecycle ordering**: the started event is emitted before budget
  warning hooks, pre-compact hooks, or remote summary work can block.
- **TUI projection**: runtime started events enter the existing
  `Compacting context...` activity state immediately.
- **Manual parity**: `/compact` emits the same started state before its
  synchronous summary call and returns to idle after the compacted notice.
- **Real interruption**: `Ctrl+C`, `Esc`, and `Ctrl+G` continue to interrupt
  while the TUI is in `Compacting context...`; the same cancellation token now
  reaches compaction hooks and the streaming DeepSeek summary request.
- **Transport-owned cancellation**: the complete DeepSeek request send and
  wait-for-response-headers future, retry delays, error-body reads, and SSE
  body reads are async and race cancellation. Cancelling drops the in-flight
  HTTP future or response so the peer observes connection close instead of
  leaving a blocking request or reader thread alive.
- **Operation-scoped HTTP client**: each async provider operation owns its
  reqwest client and uses async Hickory DNS, avoiding a global connection pool
  shared across the compatibility facade's short-lived Tokio runtimes.
- **Joined compatibility worker**: the synchronous provider API temporarily
  runs the async provider on one named worker, delivers each step to the caller
  thread through an acknowledged zero-capacity handoff, and joins the worker
  before returning on success, error, or cancellation. Callback unwind and
  facade drop cancel first and then join; the acknowledged handoff prevents a
  callback-triggered cancellation from delivering a prefetched second delta,
  including another event assembled from the same SSE frame.
- **Stream integrity**: malformed SSE JSON, invalid UTF-8, and streams that end
  without a finish reason or `[DONE]` are explicit failures. Orca retries one
  integrity failure only before any visible step has been emitted. If a visible
  partial response already reached the callback, it is not rolled back or
  replaced; the integrity error is returned directly so no duplicate delta is
  replayed.
- **Malformed known-tool recovery**: a known tool name with incomplete or
  invalid JSON arguments remains a `ToolRequest` with its raw arguments. The
  execution boundary validates those arguments before approval, hooks, task
  creation, or tool execution and returns a tool failure to the model for
  correction. Unknown tool names remain hard errors, while legacy internal
  target-only requests keep their existing compatibility path.
- **Truthful completion**: `context.compacted` is emitted only after summary
  state persistence and the post-compact hook finish, so the TUI does not return
  to idle while compaction work is still blocked.
- **Typed payloads**: started and completed compaction payloads share typed core
  structs before the existing JSONL wire serialization boundary.
- **Wire compatibility**: older `context.compacted` envelopes that contain only
  before/after message counts still project into the TUI with empty optional
  detail fields.
- **Completion detail preserved**: completed compactions still report reason,
  strategy, before/after message counts, and collapsed-message totals.

## TUI impact

Long DeepSeek summary requests no longer look like a frozen turn. Users see
`Compacting context...` for automatic soft-limit, hard-limit, prompt-too-long
recovery, and manual compaction from the beginning of the potentially slow
operation.

Compaction remains controllable while that status is visible. Interrupting a
remote DeepSeek summary closes connections stalled while waiting for response
headers or reading the response body, joins the provider worker, and lets the
turn unwind through the existing cancelled-session path. A callback-triggered
interrupt cannot receive a second delta that the worker prefetched while the
callback was still running. Invalid known-tool arguments are surfaced as a
tool failure so the model can correct the call without terminating the turn.

## Compatibility

No changes to CLI flags, slash-command syntax, model/provider configuration,
MCP configuration, hooks, workflows, npm package layout, approvals, shortcuts,
or the existing `context.compacted` payload.

The sync provider worker is a migration boundary, not the target runtime
architecture. The next refactor line introduces one-shot operation scopes and a
runtime-owned operation host, then calls the async provider directly.

The JSONL runtime event stream adds `context.compaction.started`. Existing
consumers that ignore unknown event types remain compatible, and the TUI keeps
accepting the earlier minimal `context.compacted` payload.
