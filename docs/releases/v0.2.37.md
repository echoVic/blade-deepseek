# Orca v0.2.37

Patch release: fix macOS Sequoia sandbox shell resolution failure.

## Changes

- **Fix macOS 15 sandbox shell path**: `sandbox-exec` invocations now use
  `/bin/sh` instead of bare `sh`. On macOS Sequoia (Darwin 25.x), resolving
  the bare name `sh` requires reading `/private/var/select/sh`, which the
  kernel blocks even when the seatbelt profile allows `file-read*`. This caused
  every sandboxed bash command to fail with `Error opening
  /private/var/select/sh: Operation not permitted`, triggering an unnecessary
  "Unsandboxed Shell Required" approval prompt on every tool call in full-auto
  mode. Using the absolute path bypasses the select-mechanism lookup entirely.

## Compatibility

No changes to CLI flags, TUI, server methods, tool schemas, JSONL events,
approval rules, npm package layout, or install commands. The fix applies to
both the workspace-write and read-only seatbelt profiles.
