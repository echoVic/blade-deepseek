# Orca v0.1.41

Patch release: workflow concurrency control hardening and structured failure taxonomy.

## Changes

- **Concurrency control rewrite**: `parallel()` in workflow host now uses
  `Promise.allSettled` with fail-fast semantics — the first rejection is
  re-thrown immediately while remaining tasks are awaited for cleanup.
- **Structured failure taxonomy**: new `WorkflowFailureKind` and
  `AgentFailureKind` enums classify errors into tool/MCP/token/schema
  categories, enabling targeted retry policies.
- **Concurrency metrics in evidence bundles**: evidence now tracks peak
  concurrent agents, total agent-seconds, and per-phase concurrency counts.
- **Agent retry metadata**: evidence bundles record retry attempts with
  failure kind and backoff duration for post-mortem analysis.
- **Stress-test workflow**: new `tests/fixtures/workflows/stress-evidence.js`
  exercises high-concurrency scenarios and validates metric collection.

## Compatibility

No changes to CLI flags, JSONL event names, server-mode wire shapes, npm
package layout, or install commands.
