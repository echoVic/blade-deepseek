# Orca v0.1.97

Patch release: move server permission response dispatch into a focused server
processor module.

## Changes

- **Permission processor boundary**: `server/router.rs` now delegates decoded
  permission response operations to `server/processors/permission.rs`.
- **Covered operation**: `PermissionRespond` now has a focused processor
  entrypoint while keeping the existing permission resolver implementation in
  place.
- **Architecture guardrail**: a new runtime architecture test keeps
  permission response dispatch details out of the generic router.
- **Behavior preservation**: turn/session grants, strict auto-review,
  filesystem overlays, network allow/deny overlays, and request-permission
  response events keep the same protocol semantics.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, TUI behavior, install commands, or npm package
layout.

This release continues the Codex-style request-processor refactor path. Submit
processor extraction is the remaining server processor follow-up slice.
