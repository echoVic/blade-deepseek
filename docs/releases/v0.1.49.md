# Orca v0.1.49

Patch release: resilient MCP resource discovery.

## Changes

- **Partial MCP resource listing**: `list_mcp_resources` now returns resources
  from healthy MCP servers even when another configured server fails
  `resources/list`.
- **Per-server error aggregation**: all-server resource discovery includes an
  `errors` array with the server name and failure message, so models can use
  available context while still seeing degraded MCP servers.
- **Strict filtered reads preserved**: `list_mcp_resources` with a `server`
  filter and `read_mcp_resource` keep their strict error behavior for precise
  server/resource requests.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode wire
shapes, npm package layout, install commands, MCP tool routing, or
`read_mcp_resource` arguments. All-server `list_mcp_resources` output now wraps
the resource array in `{ "resources": [...], "errors": [...] }`; server-filtered
list calls continue returning the resource array for that server.
