# Orca v0.2.18

Patch release: let DeepSeek correct an unknown or malformed function name
inside the current agent turn without granting that name executable capability
or pausing an otherwise healthy persistent Goal.

## Incident And Root Cause

During automatic Goal continuation, DeepSeek emitted
`function.name = "wc -l"` instead of calling `bash` with a command argument.
The provider parser rejected the unknown name and converted it into a terminal
`ProviderStep::Error`. That failed the whole agent turn and caused Goal mode to
pause even though the provider connection and the rest of the session were
healthy.

## Changes

- **Recoverable unknown calls**: every provider function call remains a
  `ToolRequest`. Configured external names keep `ToolName::External` and their
  declared action; unresolved `mcp__*` names keep `ToolName::Mcp`, while other
  generic unresolved names such as `wc -l` become `ToolName::External`. Every
  unresolved request receives provisional `ActionKind::Read`.
- **Provider data preserved**: the original provider call id, function name,
  and raw argument string survive both streaming and non-streaming response
  folding.
- **Side-effect boundary**: registry validation rejects an unresolved call
  before approval, hooks, task creation, or execution.
- **Matching history**: Orca records the original assistant tool call and a
  matching failed tool result containing `unknown tool: <name>`, then sends the
  result back to DeepSeek for correction inside the same agent turn.
- **TUI corrective turn**: regression coverage drives the actual TUI main-agent
  loop through an unknown `wc -l` call, the recorded failure, and a successful
  corrected response.
- **Terminal errors unchanged**: genuine transport, provider, quota,
  content-filter, truncation, and stream-integrity failures still fail the turn
  and pause automatic Goal continuation.

## Safety Boundary

Orca does not infer capability from the shape of a model-generated function
name. In particular, whitespace, shell metacharacters, and command-like text
such as `wc -l` never cause an unknown name to be coerced to `bash` or executed
as a command. Configured built-in, MCP, and external tools continue to resolve
through their registered identity, action, and schema. Unresolved MCP and
generic names retain the identity categories above but remain non-executable
and produce only a failed tool result.

## User Outcome

A single hallucinated function name no longer ends an otherwise recoverable
Goal turn. The failed call remains visible in conversation history, DeepSeek
receives the precise validation diagnostic, and it can issue a valid tool call
or answer on the next provider cycle. Actual failed, interrupted, or
approval-needed turns still stop automatic continuation.

## Compatibility

No changes to CLI flags, slash-command syntax, provider/model configuration,
MCP configuration, external-tool configuration, hooks, workflows, approvals,
shortcuts, persisted history formats, Goal record formats, or npm package
layout.

Streaming and non-streaming provider paths now share the same unknown-call
preservation behavior. Existing registered tools and known-tool argument
validation keep their prior semantics.

Install or pin the release with:


```bash

  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.18 sh
```
