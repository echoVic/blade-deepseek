# Orca v0.1.118

Patch release: continue the P0 runtime reducer split by moving turn-opening
orchestration out of `lifecycle.rs` and into a focused
`runtime_turn_opening` module.

## Changes

- **Dedicated turn-opening module**: `RuntimeTurnOpeningStep`,
  `RuntimeTurnOpeningInput`, and `RuntimeTurnOpeningResult` now live in
  `runtime_turn_opening.rs` instead of `lifecycle.rs`.
- **Grouped opening boundary**: `RuntimeTurnIterationStep` now calls opening
  through `RuntimeTurnOpeningInput`, keeping compaction, turn start, model
  routing, and steer handling behind one request-scoped boundary.
- **Smaller lifecycle surface**: `lifecycle.rs` no longer owns the opening
  orchestration layer, while it still owns the lower-level lifecycle, turn
  start, model-route, steer, and actor primitives for the next split.

## Compatibility

No changes to CLI flags, JSON protocol method names, server event names,
permission profile TOML shape, install commands, npm package layout, provider
requests, or approval response wire shapes.

This release is an internal architecture slice. Runtime behavior, model
routing, compaction, steering, tool execution, approvals, workflows, shell
sessions, and history persistence remain unchanged.
