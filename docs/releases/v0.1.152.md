# Orca v0.1.152

Patch release: install the runtime extension kernel in turn state and thread it
through the live provider/tool turn path so goal lifecycle contributors can
observe real normal-tool completions without a test-only injection path.

## Changes

- **Runtime-owned extension state**: `RuntimeTurnState` now owns the default
  `ExtensionRegistry` plus thread and turn `ExtensionData` stores.
- **Goal lifecycle installation**: each runtime turn state installs the goal
  tool lifecycle contributor, giving goal mode a live progress hook for normal
  tool execution.
- **Provider/tool turn wiring**: `run_agent_loop`, `RuntimeTurnLoopInput`,
  `RuntimeTurnIterationInput`, and `RuntimeProviderCycleInput` now pass the
  extension registry and stores into `RuntimeStepContext`, which forwards them
  into `ToolExecutionContext`.
- **Regression coverage**: focused tests prove both the default runtime state
  installs goal lifecycle extensions and normal tool turns notify that
  lifecycle through the runtime path.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, TUI
rendering, server wire shapes, app-server events, JSONL event names, thread
history, goal storage format, tool schemas, npm package layout, or install
commands.

This remains an internal architecture slice, but the extension kernel is now on
the live runtime turn path instead of being only manually injectable in tests.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.152 sh
```
