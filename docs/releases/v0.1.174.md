# Orca v0.1.174

Patch release: add an explicit server-mode read operation for long-running
`command/exec` process handles.

## Changes

- **Command exec reads**: server-mode clients can now send
  `command/exec/read` with a `processId` to actively drain a running streaming
  `command/exec` process.
- **Read acknowledgments**: successful reads return `command_exec_read` with
  the active `processId` and `status: "running"`, giving clients a request/ack
  boundary around stream polling.
- **Shared stream path**: read-triggered output reuses the existing
  `command_exec_output_delta` event, `command/exec/outputDelta` notification
  shape, output cap metadata, sandbox/network permission retry path, and
  `command_exec_completed` terminal event.
- **Streaming testability**: app-server tests can now trigger output drains
  without relying on writes, resize requests, termination, or incidental
  background polling.

## Compatibility

No changes to CLI flags, slash commands, model/provider configuration, MCP
configuration, hook schema, workflow scripts, npm package layout, install
commands, or existing JSONL event names.

Existing `command/exec`, `command/exec/write`, `command/exec/resize`,
`command/exec/terminate`, `shell/read`, `shell/capabilities`, and PTY behavior
is unchanged. `command/exec/read` is additive.

Example request:

```json
{
  "id": "cmd-read",
  "method": "command/exec/read",
  "params": {
    "processId": "process-1",
    "timeoutMs": 5000
  }
}
```

Example acknowledgment:

```json
{
  "id": "cmd-read",
  "event": "command_exec_read",
  "processId": "process-1",
  "status": "running"
}
```

Read-triggered output continues to arrive as `command_exec_output_delta` events.
If output was already available when the read request arrived, the server may
emit those deltas before the `command_exec_read` acknowledgment because active
command processes are drained before each server request is dispatched.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.174 sh
```
