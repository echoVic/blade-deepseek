# Orca v0.2.32

Feature release: keep completed DeepSeek response rows identical while live,
after approval continuation, and after thread restart or resume.

## Why

Orca previously had two authorities for a completed assistant response. Live
server and TUI projection assembled agent-message, reasoning, and proposed-plan
rows from transient deltas with static ids, while persistence wrote one combined
assistant message with a different id and shape. Reloading a thread could
therefore replace rows that the client had already observed, and a second model
response in one turn could reuse the same live ids.

This release moves identity and completion to the runtime ownership boundary.
One response receives its item ids before streaming begins, and one durable
completed-response event owns the accepted assistant result.

## Changes

- **Pre-stream identity**: every provider response reserves distinct
  agent-message, reasoning, and proposed-plan ids before its first visible
  delta, including later responses in a tool loop.
- **One durable completion**: accepted output is persisted as one typed
  `model.response.completed` event. New assistant responses are no longer also
  written as `conversation.message` records.
- **Stable approval continuation**: provider suspension, host-adopted
  background approval, persisted approval state, and continuation retain the
  response ids and owning logical turn.
- **Shared live and cold reducer**: runtime item lifecycle events, model replay,
  `thread/read`, `thread/turns/list`, and `thread/items/list` derive the same
  canonical completed objects.
- **No fabricated terminal rows**: the runtime projector no longer owns static
  text ids, completed-text accumulation, or `session.completed` synthesis.
- **Bounded compatibility**: historical combined assistant records remain
  readable through one named legacy reducer. Malformed current completion
  events fail closed instead of silently falling back.
- **Real restart proof**: the DeepSeek release gate records, cold-reads, exits,
  resumes in a second process, and compares complete persisted item-object
  prefixes plus matching internal and external ids.

## Compatibility

CLI arguments, TUI keys and flows, server request methods and response wrappers,
tool and workflow item shapes, persisted user/tool records, and existing history
files remain compatible.

`model.response.completed` is an additive semantic event. Assistant delta
payloads add opaque item identity fields while retaining their existing text.
For newly recorded responses, cold thread history now returns the existing live
`agent_message`, `reasoning`, and `plan` shapes instead of the old combined
assistant object. Legacy combined records are still projected in their original
shape and are not rewritten during normal read or resume.

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.32
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.32 sh
```
