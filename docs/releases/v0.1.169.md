# Orca v0.1.169

Patch release: move batch dispatch-window construction into the runtime-owned
sampling request state.

## Changes

- **Dispatch-window ownership**: `RuntimeSamplingRequestState` now produces
  `RuntimeToolDispatchWindow` values for batch tool dispatch.
- **Tool-turn cleanup**: `tool_turn` no longer reads raw cursor positions or
  slices `tool_requests` directly for readonly and subagent batches.
- **Forward-progress guard**: dispatch windows clamp their end index and still
  include the current request when a collector returns the current cursor,
  preventing an empty window from stalling the tool loop.
- **Regression coverage**: architecture contracts require the dispatch-window
  API, reject production raw cursor reads in `tool_turn`, and behavior tests
  cover clamped windows plus stalled-collector advancement.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice. Provider responses, normal tool
execution, permission requests, readonly batching, subagent batching, workflows,
history projection, and public release packaging keep the same user-visible
behavior as v0.1.168.
