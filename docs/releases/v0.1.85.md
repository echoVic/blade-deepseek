# Orca v0.1.85

Patch release: persist session-scoped network domain grants from `request_permissions` into server thread `command/exec`.

## Changes

- **Network permission requests**: model-visible `request_permissions` now accepts `permissions.network.domains` with permission-profile-style `allow` / `deny` entries.
- **Session grant persistence**: `permission/respond` with `scope: "session"` now stores approved network domain entries in thread metadata and live server thread state.
- **command/exec inheritance**: later thread-bound `command/exec` calls merge those session network domains into the managed proxy policy, so allowlist misses keep returning Codex-style `blocked-by-allowlist` diagnostics.
- **Harness coverage**: the mock provider now has a network permission request path, and server contract coverage proves the full request / response / later command proxy loop.

## Compatibility

No changes to command/exec method names, sandbox policy names, permission profile TOML shape, network proxy environment variables, CLI flags, JSONL event names, TUI behavior, npm package layout, or install commands.

This release seeds interactive network ask flows for server threads. Automatic ask-on-network-block behavior remains follow-up work.
