# Orca v0.1.148

Patch release: group async subagent worker inputs behind focused context
objects so the worker launch and worker-entrypoint APIs keep shrinking without
changing runtime behavior.

## Changes

- **Grouped worker input**: `AsyncSubagentWorkerInput` now carries the worker
  config, parent cwd, child cwd, task session id, agent id, request, child
  depth, and optional worktree handoff for `run_async_subagent_worker`.
- **Injectable worker context**: `AsyncSubagentWorkerContext` now groups the
  worker input with the child executor used by
  `run_async_subagent_worker_with_executor`.
- **Grouped async launch**: `AsyncSubagentLaunchContext` now carries the parent
  runtime config, cwd, original tool request, subagent request, depth, and task
  registry for `launch_async_subagent`.
- **Shorter spawn boundary**: async worker process spawning now receives a
  focused spawn context instead of a long config/cwd/session/request/worktree
  argument list.
- **Ownership coverage**: the async worker architecture test now locks the
  context structs, context-taking function signatures, and absence of
  `#[allow(clippy::too_many_arguments)]` in `subagent_async_worker.rs`.

## Compatibility

No changes to CLI flags, worker subprocess JSON shape, task registry records,
subagent tool schema, async completion payloads, worktree handoff behavior,
sync subagent execution, workflow child agent calls, TUI async subagent
rendering, MCP routing, approval policy semantics, shell session protocol,
command/exec wire shape, conversation storage, history storage, npm package
layout, or install commands.

This release is an internal ownership and API-shape slice. Existing CLI and
runtime async subagent flows keep the same behavior while the worker module no
longer relies on long-argument helper APIs or clippy allowances.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.148 sh
```
