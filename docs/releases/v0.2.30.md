# Orca v0.2.30

Feature release: complete the production TUI migration onto the runtime host so
DeepSeek turns, interactive waits, background work, and shutdown share one
traceable cancellation and join boundary.

## Why

The TUI previously wrapped the canonical runtime with its own provider/tool
loop, session owner, cancellation state, background supervisor, and workflow
watchers. Those paths could preserve visible behavior, but they left two
long-lived owners for active operations, task terminals, usage, and cleanup.
Adding another cancellation check would not fix that ownership defect.

This release removes the parallel execution path. `RuntimeHost` and its
`ThreadActor` now own the live thread, every foreground operation, admitted
background providers and workflows, terminal publication, and shutdown joins.

## Changes

- **One canonical turn**: TUI, server, and headless execution use the same
  `ThreadTurnExecutor` provider/tool/compaction/hook kernel and typed runtime
  events.
- **Reliable interruption**: `Ctrl-C` targets the exact hosted operation. The
  runtime cancels and joins that DeepSeek generation before publishing its
  terminal, and the next submit receives a fresh operation.
- **Owned interactions**: approval, permission, user-input, and MCP elicitation
  waits use an owned broker fenced by operation id, request id, and interaction
  kind. Interrupt and shutdown wake waiters without borrowing the TUI action
  receiver.
- **Owned background work**: provider suspension and workflow launches return
  typed handoffs. RuntimeHost applies bounded admission, cancellation, join,
  exactly-one terminal publication, usage settlement, and shutdown cleanup.
- **Typed saved workflows**: production `/workflow` actions submit
  `HostedWorkflowRequest` values to the host and no longer start a detached TUI
  watcher.
- **Old owners removed**: the TUI agent runner, provider/tool/workflow/subagent
  execution modules, local operation cancellation path, and `TuiTaskSupervisor`
  are deleted instead of retained as compatibility layers.
- **Clear task identity**: runtime lifecycle operation ids, persisted server
  turn ids, and `TaskRegistry` task ids are separate, preventing a persisted
  `turn-1` request from settling the wrong task.
- **Quieter Goal status**: repeated idle refreshes collapse an unchanged Goal
  notice instead of appending duplicates.

## Compatibility

CLI arguments, TUI keys and user flows, slash commands, server method names,
JSONL event names, persisted conversations and tasks, workflow scripts, npm
package layout, model selection, and DeepSeek request semantics remain
compatible. The runtime changes are internal ownership and additive typed-event
improvements.

```bash
cargo fmt --all -- --check
git diff --check

npm --prefix site run build
npm --prefix site run check:seo
node scripts/release/test-stage-npm.mjs
node scripts/release/test-verify-published.mjs
node scripts/release/test-real-api-e2e.mjs
node scripts/release/real-api-e2e.mjs --max-budget 0.02 --timeout-ms 300000
```

Production PTY validation streams a live `deepseek-v4-flash` response, updates
context state, interrupts a long live stream, accepts a fresh submit, and
restores cursor, keyboard, paste, mouse, and alternate-screen modes.

Post-publish verification:

```bash
node scripts/release/verify-published.mjs --version 0.2.30 --repo echoVic/blade-deepseek --package @blade-ai/orca --bin orca
```

## Upgrade

Install or pin the release with:

```bash
npm install -g @blade-ai/orca@0.2.30
```

or:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.2.30 sh
```
