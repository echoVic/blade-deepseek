# Orca v0.1.132

Patch release: continue the Codex/package-3-inspired runtime boundary cleanup by
grouping internal model-visible bash execution inputs behind focused context
types.

## Changes

- **Grouped sandbox execution context**: model-visible bash sandbox execution now
  receives `RuntimeBashSandboxContext`, keeping permission-profile sandboxing,
  permission request/retry state, cancellation, task-registry handoff, output
  truncation, and diagnostics in one explicit internal contract.
- **Grouped one-shot shell context**: non-shell-session bash spawning now receives
  `RuntimeBashOnceContext`, removing another long helper argument list from
  `runtime_bash.rs`.
- **No helper escape hatch**: `runtime_bash.rs` no longer needs an internal
  `clippy::too_many_arguments` allow for its sandbox/one-shot execution helpers.
- **Behavior-compatible flow**: shell-session invocation, sandbox policy
  selection, managed network proxy retries, filesystem sandbox-denial retries,
  task registry updates, output truncation, cancellation, and diagnostics keep
  the same runtime behavior.

## Compatibility

No changes to CLI flags, JSONL event names, bash tool input/output shape, shell
session protocol, command/exec behavior, permission-profile configuration,
network allow/deny semantics, filesystem sandbox grants, MCP routing, external
tool routing, conversation storage, history storage, workflows, server protocol,
TUI rendering, npm package layout, or install commands.

This release is an internal architecture slice. Model-visible bash execution
keeps the same public behavior while the runtime boundary becomes easier to keep
small during the next shell/session and permission-retry refactors.
