# Orca v0.1.50

Patch release: MCP resource template discovery.

## Changes

- **MCP resource templates**: added the read-only
  `list_mcp_resource_templates` tool so models can discover parameterized MCP
  resources before choosing a concrete URI.
- **Transport support**: stdio and SSE MCP transports now call
  `resources/templates/list` and parse `resourceTemplates` with server,
  `uriTemplate`, name, description, and optional `mimeType` metadata.
- **Partial failure reporting**: all-server template discovery returns
  `{ "resourceTemplates": [...], "errors": [...] }`, preserving healthy server
  templates while surfacing per-server failures.
- **Strict server filters preserved**: `list_mcp_resource_templates` with a
  `server` filter returns that server's template array or fails strictly, matching
  the filtered behavior of `list_mcp_resources`.

## Compatibility

No changes to CLI flags, config file shape, JSONL event names, server-mode wire
shapes, npm package layout, install commands, MCP tool routing, existing
`list_mcp_resources`, or `read_mcp_resource` arguments. The new tool is additive
and read-only.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.50 sh
```
