# Orca v0.1.63

Patch release: shared command lifecycle item projection for realtime server
streams.

## Changes

- **Shared command items**: realtime app-server `commandExecution` item
  lifecycle events now use `tool_item_projection` builders instead of
  hand-written JSON shapes in `server_runtime`.
- **Protocol-boundary cleanup**: `ServerRequestWriter` now stores command tool
  state while shared builders own started/completed command item schema,
  matching the existing MCP, dynamic, text, fileChange, and workflow projection
  boundary.
- **Projection coverage**: unit tests lock commandExecution started,
  completed-success, and failed-diagnostic shapes, including `aggregatedOutput`,
  `exitCode`, `truncated`, and the absence of legacy `output`.
- **Compatibility coverage**: server writer tests still verify successful
  command lifecycle streams and failed command diagnostics end to end.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release is an internal projection-boundary refactor for realtime
`commandExecution` lifecycle items.
