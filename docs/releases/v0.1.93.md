# Orca v0.1.93

Patch release: move synchronous server thread query and metadata dispatch into
the first focused server processor module.

## Changes

- **Thread processor boundary**: `server/router.rs` now delegates decoded
  thread query and metadata operations to `server/processors/thread.rs`.
- **Covered operations**: `ThreadRead`, `ThreadList`, `ThreadSearch`,
  `ThreadTurnsList`, `ThreadItemsList`, and `ThreadMetadataUpdate` now share
  the focused thread processor entrypoint.
- **Architecture guardrail**: a new runtime architecture test keeps these
  thread dispatch details out of the generic router.
- **Behavior preservation**: thread projection, metadata updates, live-thread
  fallback reads, and finished-turn reclamation are preserved.

## Compatibility

No changes to CLI flags, JSON protocol methods, server event names, permission
profile TOML shape, TUI behavior, install commands, or npm package layout.

This release continues the Codex-style request-processor refactor path started
in v0.1.92. Submit, turn-control, permission, shell, and command/exec processor
extractions remain follow-up slices.
