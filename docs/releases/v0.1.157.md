# Orca v0.1.157

Patch release: route permission overlay mutation through the runtime-owned turn
reducer.

## Changes

- **Permission reducer branch**: `RuntimeTurnReducer` now owns a permission
  reducer branch for requesting permission grants and merging turn overlays.
- **Request-permissions routing**: model-visible `request_permissions` now
  updates its overlay through the reducer instead of directly mutating
  `TurnPermissionOverlay`.
- **Bash escalation routing**: bash network and filesystem auto-escalation keep
  the same permission request / retry behavior while routing overlay mutation
  through the reducer.
- **Router overlay transfer**: `RuntimeToolRouter` now merges runtime actor
  permission overlay state through the reducer after `request_permissions`
  completes.
- **Regression coverage**: tests prove allowed permission responses still merge
  filesystem roots, network domains, and `strictAutoReview`, and boundary tests
  prevent the direct overlay mutation calls from returning to runtime-special,
  runtime-bash, and router call sites.

## Compatibility

No changes to CLI flags, slash commands, approval policy semantics, permission
request wire shapes, TUI display layout, server events, JSONL event names,
thread metadata, tool schemas, npm package layout, or install commands.

This is an internal architecture slice in the Codex/package-3 direction:
runtime-owned reducers now handle tool finish state, runtime directives, and
permission overlay mutation while preserving behavior-level interfaces.
