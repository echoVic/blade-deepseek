# Orca v0.1.150

Patch release: seed a Codex-inspired runtime extension contributor kernel so
goal, memory, task, and tool lifecycle behavior can move out of the main
runtime loop behind typed per-scope extension data and ordered contributors.

## Changes

- **Extension data scopes**: `orca-runtime` now exposes `ExtensionData`, a
  typed attachment map keyed by runtime scope id, so thread and turn features
  can own private state without expanding long runtime structs.
- **Lifecycle contributor registry**: `ExtensionRegistryBuilder` and
  `ExtensionRegistry` now support ordered tool lifecycle contributors with
  explicit tool start/finish inputs and outcome metadata.
- **Goal lifecycle seed**: the goal module can install a tool lifecycle
  contributor that records completed non-`update_goal` tool attempts in
  thread-scoped goal progress state, establishing the first real migration
  path for Codex-style goal extension behavior.
- **Architecture plan**: the feature round is captured in
  `docs/superpowers/plans/2026-07-06-extension-contributor-kernel.md` so
  follow-up slices can continue migrating runtime hooks through the same
  contributor boundary.
- **Roadmap alignment**: the production roadmap now records the extension
  contributor kernel as the next cross-cutting runtime boundary after event
  projection.

## Compatibility

No changes to CLI flags, slash commands, goal storage format, `update_goal`
tool schema, JSONL runtime event names, app-server wire event shapes, thread
history, command/exec protocol, shell session protocol, workflow behavior, MCP
routing, external tool routing, approval policy semantics, npm package layout,
or install commands.

This release is an internal architecture slice. Existing CLI, TUI, server, and
app-server flows keep the same behavior while the runtime gains a narrow
extension point for future goal, memory, task, and tool lifecycle migration.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.150 sh
```
