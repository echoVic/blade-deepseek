# Orca v0.1.57

Patch release: shared MCP and dynamic tool started-item projection builders.

## Changes

- **Shared started items**: realtime app-server streams and persisted thread
  projections now use the same `tool_item_projection` builders for
  `mcpToolCall` and `dynamicToolCall` started items.
- **Schema drift guardrail**: the ownership test now prevents
  `server_runtime` and `thread_store` from reintroducing separate MCP/dynamic
  started-item builders.
- **Behavior coverage**: projection unit tests lock the public started-item
  shape for MCP and dynamic tool calls, including arguments, null result/error
  slots, and dynamic `contentItems` / `success` placeholders.
- **Compatibility preserved**: server streams and persisted history keep the
  same public item field names while construction moves behind the shared
  projection boundary.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm package
layout, install commands, JSONL event names, or public item field names. This
release is an internal projection-boundary refactor for first-class
`mcpToolCall` and `dynamicToolCall` started items.
