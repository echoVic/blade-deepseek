# Orca v0.1.70

Patch release: the TUI now splits finished transcript history from the live bottom viewport.

## Changes

- **Native terminal scrollback**: Settled TUI messages now flush into the terminal's native scrollback through ratatui inline viewport insertion, so finished history can use normal terminal scrolling and text selection.
- **Live viewport boundary**: The bottom pane now renders only still-mutable content: streaming assistant output, live tool rows, the plan panel, composer, status bar, and modal/full-panel states.
- **Immutable history semantics**: Tool-output expansion is limited to live tool calls; flushed tool output is committed in expanded form. Backtrack keeps printed scrollback and appends a reset marker instead of trying to erase immutable terminal history.
- **Dynamic inline height**: The inline viewport grows for composer, plan, goal, popup, approval, help, setup, session picker, workflow, and agent views, while capping live streaming output to avoid rebuild churn.
- **Clear-screen behavior**: Clearing the TUI now resets both Orca's message state and the terminal's native scrollback.

## Compatibility

No changes to CLI flags, config file shape, server-mode methods, npm install
commands, JSONL event names, public realtime event names, or npm package layout.

The TUI behavior changes intentionally:

- Finished history is no longer re-rendered inside Orca's live pane after it is flushed.
- `e` can expand only live tool output; finalized tool output is fixed in scrollback.
- Mouse wheel scrolling is left to the terminal's native scrollback rather than Orca's in-app history pane.

## Install

```bash
npm install -g @blade-ai/orca
orca --version
```

Direct binary install:

```bash
curl -fsSL https://orcaagent.dev/install.sh | sh
```

Pin a specific version:

```bash
curl -fsSL https://orcaagent.dev/install.sh | \
  INSTALL_DIR=/usr/local/bin ORCA_VERSION=0.1.70 sh
```
